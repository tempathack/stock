## Codebase Patterns

- Use `imagePullPolicy: Never` for local Minikube development (no registry needed)
- K8s secrets cannot be referenced cross-namespace — copy with: kubectl get secret <name> -n storage -o yaml | kubectl apply -n <target>
- yfinance >= 1.0 returns MultiIndex columns — use `.droplevel(1)` after `.download()`
- FastAPI endpoints return 502 (not 500) for upstream Yahoo Finance or Kafka failures
- `/predict/bulk` registered before `/{ticker}` in router to prevent path variable capture
- Playwright tests all require `test.describe.configure({ mode: 'serial' })` — parallel mode overwhelms Vite dev server
- SHAP imports guarded with `_shap_available` flag — numba 0.64 requires numpy<2.0, incompatible with test env
- All Python code uses type hints and docstrings on public functions
- Backend pytest: add `-p no:logfire` flag to avoid broken logfire plugin
- prd.json and progress.txt live in scripts/ralph/ relative to repo root
- TimescaleDB: use `CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE`
- Kafka consumers: idempotent upsert on (ticker, timestamp) primary key
- Model serving directory: pipeline.pkl + features.json + metadata.json (flat layout)
- SHAP: TreeExplainer for tree/boosters, LinearExplainer for linear family, KernelExplainer fallback
- Drift detectors: DataDriftDetector (KS-test+PSI), PredictionDriftDetector, ConceptDriftDetector
- All three detectors share DriftResult dataclass with severity (none/low/medium/high)

- Created `stock-prediction-platform/scripts/test-production.sh` — single entry point for all production E2E tests
- Script supports flags: `--skip-infra`, `--skip-app`, `--report`

- Step 2: Port-forwards — kills stale pf processes on target ports, starts 6 core + 2 optional (KFP, K8s dashboard); tracks PIDs in PF_PIDS array + /tmp/pf-prod-*.pid files; trap cleanup on EXIT
- Step 3: Health-check loop — curl retry (3s interval, up to 60s) for each service; prints ready/unreachable; optional services allowed to fail
- Step 4: Environment exports — BASE_API_URL, GRAFANA_URL, PROMETHEUS_URL, MINIO_URL, KUBEFLOW_URL, K8S_DASHBOARD_URL; prints KUBERNETES_DASHBOARD_TOKEN generation instructions if not set
- Step 5: Runs app tests (playwright.production.config.ts) then infra tests (playwright.infra.config.ts); each in separate npx calls; captures exit codes
- Step 6: Prints summary with pass/fail per suite; offers HTML report opening via --report flag
- Cleanup trap kills all PF_PIDS and removes /tmp/pf-prod-*.pid files on exit
- **Learnings for future iterations:**
  - Service names in actual cluster: `svc/frontend` (not `svc/frontend-service`), `svc/stock-api` (not `svc/stock-api-service`) — match deploy-all.sh names
  - Grafana mapped to port 3001 (host) → 3000 (pod) to avoid conflict with frontend on 3000
  - MinIO console (9001) and API (9000) need separate port-forward entries; deploy-all.sh only does console
  - `PF_PIDS` array approach is cleaner than /tmp PID files for in-process tracking
  - `bash -n` validates shell syntax; confirmed script runs gracefully without minikube missing
