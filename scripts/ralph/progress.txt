## Codebase Patterns

- Use `imagePullPolicy: Never` for local Minikube development (no registry needed)
- K8s secrets cannot be referenced cross-namespace — copy with: kubectl get secret <name> -n storage -o yaml | kubectl apply -n <target>
- yfinance >= 1.0 returns MultiIndex columns — use `.droplevel(1)` after `.download()`
- FastAPI endpoints return 502 (not 500) for upstream Yahoo Finance or Kafka failures
- `/predict/bulk` registered before `/{ticker}` in router to prevent path variable capture
- Playwright tests all require `test.describe.configure({ mode: 'serial' })` — parallel mode overwhelms Vite dev server
- SHAP imports guarded with `_shap_available` flag — numba 0.64 requires numpy<2.0, incompatible with test env
- All Python code uses type hints and docstrings on public functions
- Backend pytest: add `-p no:logfire` flag to avoid broken logfire plugin
- prd.json and progress.txt live in scripts/ralph/ relative to repo root
- TimescaleDB: use `CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE`
- Kafka consumers: idempotent upsert on (ticker, timestamp) primary key
- Model serving directory: pipeline.pkl + features.json + metadata.json (flat layout)
- SHAP: TreeExplainer for tree/boosters, LinearExplainer for linear family, KernelExplainer fallback
- Drift detectors: DataDriftDetector (KS-test+PSI), PredictionDriftDetector, ConceptDriftDetector
- All three detectors share DriftResult dataclass with severity (none/low/medium/high)
