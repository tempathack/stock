## Codebase Patterns

- Use `imagePullPolicy: Never` for local Minikube development (no registry needed)
- K8s secrets cannot be referenced cross-namespace — copy with: kubectl get secret <name> -n storage -o yaml | kubectl apply -n <target>
- yfinance >= 1.0 returns MultiIndex columns — use `.droplevel(1)` after `.download()`
- FastAPI endpoints return 502 (not 500) for upstream Yahoo Finance or Kafka failures
- `/predict/bulk` registered before `/{ticker}` in router to prevent path variable capture
- Playwright tests all require `test.describe.configure({ mode: 'serial' })` — parallel mode overwhelms Vite dev server
- SHAP imports guarded with `_shap_available` flag — numba 0.64 requires numpy<2.0, incompatible with test env
- All Python code uses type hints and docstrings on public functions
- Backend pytest: add `-p no:logfire` flag to avoid broken logfire plugin
- prd.json and progress.txt live in scripts/ralph/ relative to repo root
- TimescaleDB: use `CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE`
- Kafka consumers: idempotent upsert on (ticker, timestamp) primary key
- Model serving directory: pipeline.pkl + features.json + metadata.json (flat layout)
- SHAP: TreeExplainer for tree/boosters, LinearExplainer for linear family, KernelExplainer fallback
- VITE_API_URL must point to localhost:8010 (Docker maps host:8010 → container:8000) — NOT 8000
- Flink restart: use kubectl patch flinkdeployment <name> -n flink --type=merge -p '{"spec":{"restartNonce":1}}'
- strimzi-cluster-operator and kafka-entity-operator are prone to OOMKill — set memory >= 512Mi
- MinIO restarts are OOM-related — set memory limit >= 512Mi, request >= 256Mi
- Redis needs --maxmemory 256mb --maxmemory-policy allkeys-lru to prevent OOM cycling
- sentiment-stream FIXED: ARRAY_LENGTH→CARDINALITY, scan.startup.mode→'latest-offset', minio-secrets→minio-s3-credentials
- daily-feature-store cron: 2x consecutive failures — investigate before next run at 18:00 weekdays
- daily-drift cron: newly added, first run scheduled at 22:00 weekdays — monitor first execution
- FastAPI CORSMiddleware MUST be added last (outermost) — adding it first causes CORS headers to be missing from responses because BaseHTTPMiddleware layers interfere. Order: RequestContextMiddleware → RateLimitMiddleware → CORSMiddleware (last added = outermost)
- kubectl port-forward drops when target pod is restarted (rollout restart) — must restart port-forward after any deployment rollout
- Backend pytest fails with ModuleNotFoundError for 'elasticsearch' in test environment — pre-existing, not our bug
- Playwright MCP browser can crash and become stuck in "Target page, context or browser has been closed" state — use headless chromium CLI as fallback: /home/tempa/.cache/ms-playwright/chromium-1217/chrome-linux64/chrome --headless --screenshot=out.png --virtual-time-budget=5000 <url>

- Rebuilt image in Minikube context, delete+reapply FlinkDeployment (rollout restart not sufficient — operator reconciles back to cached spec)
- Verified: sentiment-stream shows RUNNING/STABLE
- **Learnings for future iterations:**
  - Flink SQL valid array length function is `CARDINALITY()`, NOT `ARRAY_LENGTH()`
  - Valid Kafka scan.startup.mode values: `earliest-offset`, `latest-offset`, `group-offsets`, `timestamp`, `specific-offsets` — NOT `'latest'` or `'earliest'`
  - `kubectl rollout restart` is insufficient to update Flink jobs — must delete and reapply FlinkDeployment
  - To debug PyFlink "Python process exits with code: 1": `kubectl exec -n flink <pod> -- /usr/bin/python3 /opt/flink/usrlib/<script>.py` to get the actual Python traceback
  - All Flink deployments should use `minio-s3-credentials` (not `minio-secrets`) for S3 checkpoint access

## 2026-04-03 - FIX-009: Fix Analytics OLAP Candle Chart — ResponsiveContainer renders at -1x-1px
- Root cause: ResponsiveContainer initializes containerWidth/containerHeight to -1 (Recharts v3 default). When height="100%" (a percentage), Recharts uses containerHeight (-1) for calculatedHeight — firing the validateWidthHeight warning on first render before ResizeObserver fires.
- Fix applied to OLAPCandleChart.tsx: changed `height="100%"` → `height={280}`, added `minWidth={0} minHeight={240}` to ResponsiveContainer, added `minHeight: 240` to wrapper Box
- Also fixed StreamLagMonitor.tsx which had identical issue: changed `height="100%"` → `height={220}`, added `minHeight={180}` to ResponsiveContainer, added `minHeight: 180` to wrapper Box
- Files modified: `services/frontend/src/components/analytics/OLAPCandleChart.tsx`, `services/frontend/src/components/analytics/StreamLagMonitor.tsx`
- Verified via Playwright: 0 Recharts warnings, OLAP chart container visible with correct dimensions ($145-$180 price axis)
- **Learnings for future iterations:**
  - Recharts v3 ResponsiveContainer starts at initialDimension {width:-1, height:-1}. If height="100%", calculatedHeight=-1 on first render → warning fires. Fix: pass explicit pixel height (number) so Recharts uses Number(280) instead of containerHeight.
  - When width="100%" but height={280}: Recharts check is `calculatedWidth>0 || calculatedHeight>0` → `(-1>0)||(280>0)` = true → warning does NOT fire. Width measurement still uses ResizeObserver but height is bypassed.
  - StreamLagMonitor also had the same ResponsiveContainer issue — always check all Recharts components on the page, not just the one in the AC.
  - Recharts default minWidth is 0 (not undefined), so seeing minWidth(0) in the warning does NOT mean you set it — it's the default.
