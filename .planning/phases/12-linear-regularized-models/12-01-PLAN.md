## Steps

### 1. Implement `ml/evaluation/metrics.py`

Six metric functions + an aggregate helper:

- `compute_r2(y_true, y_pred) -> float` — sklearn `r2_score`
- `compute_mae(y_true, y_pred) -> float` — sklearn `mean_absolute_error`
- `compute_rmse(y_true, y_pred) -> float` — `sqrt(mean_squared_error)`
- `compute_mape(y_true, y_pred) -> float` — mean(|y_true - y_pred| / |y_true|) * 100, with zero-guard
- `compute_directional_accuracy(y_true, y_pred) -> float` — % where sign(pred) == sign(actual), as percentage
- `compute_fold_stability(fold_rmses: list[float]) -> float` — std of RMSE across folds
- `compute_all_metrics(y_true, y_pred) -> dict` — returns dict with all 5 point metrics

### 2. Implement `ml/evaluation/cross_validation.py`

- `create_time_series_cv(n_splits=5) -> TimeSeriesSplit` — wrapper returning configured splitter
- `walk_forward_evaluate(model, X, y, cv, metrics_fn) -> dict` — runs walk-forward CV, returns per-fold + aggregated metrics

### 3. Update `ml/evaluation/__init__.py`
- Export all public functions

### 4. Write tests

**test_metrics.py:**
- Known-value tests for each metric
- Edge cases: perfect predictions, all-zero true values (MAPE guard), single sample

**test_cross_validation.py:**
- Verify n_splits=5
- Verify no shuffling (train indices < test indices for every fold)
- Verify walk_forward_evaluate returns correct structure
