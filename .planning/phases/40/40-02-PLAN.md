<migration_notes>
## Query Parameter Style Change

psycopg2 uses `%s` positional parameters:
```python
cur.execute("SELECT * FROM t WHERE ticker = %s", (ticker,))
```

SQLAlchemy text() uses `:name` named parameters:
```python
session.execute(text("SELECT * FROM t WHERE ticker = :ticker"), {"ticker": ticker})
```

All query strings must be converted from `%s` → `:param_name` format.

## Result Access Pattern Change

psycopg2 returns tuples:
```python
cols = [desc[0] for desc in cur.description]
rows = cur.fetchall()
return [dict(zip(cols, row)) for row in rows]
```

SQLAlchemy returns Row objects with .mappings():
```python
result = await session.execute(text(query), params)
return [dict(row) for row in result.mappings().all()]
```

## Session Lifecycle

Sessions are injected by FastAPI's dependency system and automatically closed by get_async_session().
Service functions must NOT create or close sessions — they receive a session and use it.

## Fallback Strategy

Router functions that call DB service functions should handle the case where the session
dependency itself might fail (db.py returns None-yielding session on engine init failure).
The existing pattern of "try DB, fall back to file" is preserved.
</migration_notes>
