    - Module-level docstring: "Backtest service — join historical predictions with actual prices and compute accuracy metrics."

    # --- get_backtest_data(ticker, start_date, end_date, horizon, model_id) → dict | None ---
    - Async function that queries predictions joined with ohlcv_daily
    - SQL query joins predictions p with ohlcv_daily o:
      - ON o.ticker = p.ticker AND o.date = p.predicted_date
      - WHERE p.ticker = :ticker
      - AND p.predicted_date >= :start_date AND p.predicted_date <= :end_date
      - Optional: AND p.horizon = :horizon (if horizon parameter provided)
      - Optional: AND p.model_id = :model_id (if model_id parameter provided)
      - If neither horizon nor model_id specified, join model_registry to get is_active=true model
      - ORDER BY p.predicted_date ASC
    - Returns None if get_engine() is None (DB not initialized)
    - Returns None if query returns no rows
    - Extracts actual_price (o.close), predicted_price (p.predicted_price), date (p.predicted_date)
    - Computes per-point error and error_pct
    - Computes aggregate metrics using numpy:
      - rmse: sqrt(mean(errors^2))
      - mae: mean(|errors|)
      - mape: mean(|error| / |actual|) * 100, handling zero actuals
      - directional_accuracy: percentage of consecutive-point pairs where predicted direction matches actual direction
      - r2: 1 - SS_res/SS_tot
    - Fetches model_name from model_registry table (or includes in join)
    - Returns dict matching BacktestResponse shape

    # --- _compute_directional_accuracy(actuals, predictions) → float ---
    - Helper function: computes % of periods where sign(pred[i]-pred[i-1]) matches sign(actual[i]-actual[i-1])
    - Returns 0.0 if fewer than 2 data points

    # --- _compute_r2(actuals, predictions) → float ---
    - Helper function: 1 - sum((actual-pred)^2) / sum((actual-mean(actual))^2)
    - Returns 0.0 if SS_tot is zero (prevents division by zero)

<objective>
Create the /backtest/{ticker} API endpoint that joins the predictions table with
ohlcv_daily to produce an actual-vs-predicted time series with accuracy metrics.

Purpose: FENH-04 (GET /backtest/{ticker}?start=...&end=... endpoint returning
actual vs predicted series). The endpoint accepts optional date range, horizon,
and model_id parameters. It returns a BacktestResponse containing per-point
series data (date, actual, predicted, error) and aggregate metrics (RMSE, MAE,
MAPE, directional accuracy, R²). The service layer computes all metrics in-memory
from the DB result set using numpy — no dependency on the ML evaluation module
to avoid coupling the API service to the ML package.
</objective>
