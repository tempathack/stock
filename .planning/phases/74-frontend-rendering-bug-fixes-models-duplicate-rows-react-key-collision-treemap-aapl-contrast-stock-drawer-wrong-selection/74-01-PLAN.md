<objective>
Fix two JavaScript logic bugs causing incorrect UI state in the Models and Dashboard pages.

Bug 1 â€” Duplicate model rows: `ModelComparisonTable` uses `getRowId = row => "${model_name}-${scaler_variant}-${version ?? ""}"`. When the API returns multiple rows where `version` is `null`, every such row resolves to the same key (e.g. `"RandomForest-standard-"`). MUI DataGrid deduplicates by row ID, causing rows to disappear and appear as duplicates in the visible render. Fix: append the array index as a guaranteed-unique suffix.

Bug 2 â€” Wrong stock in drawer: `handleSelect` in `Dashboard.tsx` is an inline arrow function recreated on every render. It is passed as `onSelectTicker` to `MarketTreemap`, which passes it into Recharts' `content` renderer via a `useCallback` that lists `onSelectTicker` as a dependency. Because `handleSelect` changes reference every render, the Recharts Treemap content closure can capture a stale reference to an old `handleSelect` instance, firing it on click and sometimes activating the previously-selected ticker rather than the one clicked. Fix: wrap `handleSelect` in `useCallback` so the reference is stable.

Purpose: Both are silent logic bugs that corrupt user-visible state. The models table may hide trained models from view. The stock drawer may show the wrong stock analysis.
Output: `ModelComparisonTable.tsx` with collision-proof `getRowId`, `Dashboard.tsx` with memoised `handleSelect`.
</objective>
