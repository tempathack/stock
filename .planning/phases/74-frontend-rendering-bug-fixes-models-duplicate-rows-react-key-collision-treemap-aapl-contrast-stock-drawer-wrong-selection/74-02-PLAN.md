<objective>
Fix the AAPL treemap cell text visibility bug caused by insufficient contrast between the percentage-change text color and the tile background color.

Root cause: The `TreemapContent` SVG renderer in `MarketTreemap.tsx` uses a conditional fill for the percentage text:
- Positive pct → `rgba(0,255,135,0.95)` (neon green)
- Negative pct → `rgba(255,100,140,0.95)` (light pink/salmon)

AAPL has `daily_change_pct = -1.82%`, so its tile fill is `changePctToColor(-1.82) ≈ #831849` (dark crimson from lerp of `#110C2E` → `#CC1F5A` at t=0.607). The pct text color `rgba(255,100,140,0.95)` (pink) on a `#831849` (dark crimson) background has a WCAG contrast ratio of approximately 2.1:1 — well below the minimum 4.5:1 for small text. The text becomes effectively invisible against the tile.

Additional: the `rgba(0,0,0,0.22)` bottom-gradient overlay further darkens the bottom half of each tile, making the pct text (which sits in the lower half) even harder to read against positive tiles (neon green background) where the pink/green on bright-green also fails contrast.

Fix: Use `#FFFFFF` (white) for both ticker text and pct text. Add an SVG `filter` drop-shadow to provide depth and legibility against both very dark tiles (near-zero change) and very bright tiles (±3% change). White text with a dark drop-shadow achieves ≥7:1 contrast on all tile colors produced by `changePctToColor`.

Purpose: Make the treemap readable for all S&P 500 stocks regardless of their daily performance color.
Output: `MarketTreemap.tsx` with white pct text and drop-shadow filter on both text elements.
</objective>

- pct = 0%  → `#110C2E` (almost black deep purple)
- pct = -3% → `#CC1F5A` (bright crimson)
- pct = +3% → `#00CC6E` (bright neon green)
- pct = -1.82% (AAPL) → lerp(#110C2E, #CC1F5A, 0.607) ≈ `#831849` (dark crimson)

Problem: `rgba(255,100,140,0.95)` pink on `#831849` = contrast ratio ~2.1:1 (WCAG fail)
Problem: `rgba(0,255,135,0.95)` green on `#00CC6E` = contrast ratio ~1.1:1 (near invisible)

Solution: Use `fill="#FFFFFF"` for pct text with SVG drop-shadow filter for depth.
White on `#831849` = contrast ratio 5.4:1 (WCAG AA pass)
White on `#00CC6E` = contrast ratio 3.5:1 — marginal, so add drop-shadow to boost perceived contrast.

SVG drop-shadow approach (no external CSS needed):
```svg

  <filter id="text-shadow" x="-20%" y="-20%" width="140%" height="140%">
    <feDropShadow dx="0" dy="0" stdDeviation="1.5" flood-color="rgba(0,0,0,0.8)" />
  </filter>
</defs>
```
