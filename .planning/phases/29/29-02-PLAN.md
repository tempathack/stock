**Chart — dual-axis ComposedChart:**

  - `&lt;Line dataKey="directionalAccuracy"&gt;` — stroke `#00d4aa` (profit), 2px, yAxisId="right"

  - Date, RMSE (4 decimals), MAE (4 decimals), DA (percentage with 1 decimal)

- Cartesian grid: `stroke="#2a2a4a"` (border colour), `strokeDasharray="3 3"`

**Props:**
```typescript

  status: RetrainStatus;
}
```

**Layout:**
- Outer wrapper: `bg-bg-surface rounded-lg border border-border p-5`
- Title row: "Retraining Status" + status badge:
  - If `status.isRetraining` → pulsing amber badge "Retraining…" with CSS animation
  - If `status.lastRetrainDate` → green badge "Up to Date"
  - Else → gray badge "Never Retrained"

**Content sections:**

**Section 1 — Last retrain info:**
- "Last retrained: {formatted date}" or "Never retrained" if null
- Date formatted as "Mar 20, 2026 at 08:30 UTC"

**Section 2 — Model comparison (if oldModel and newModel present):**
- Two-column card layout: "Previous Model" vs "Current Model"
- Each column:
  - Model name in `text-sm font-medium text-text-primary`
  - RMSE value: `text-lg font-mono` — red tint for old, green tint for new (if improved)
  - MAE value: `text-sm font-mono text-text-secondary`
- Between columns: arrow icon (→) or "vs" separator

**Section 3 — Improvement metric:**
- Large improvement percentage: `text-2xl font-bold`
  - Green (`text-profit`) if positive improvement
  - Red (`text-loss`) if negative (regression)
- Label: "RMSE Improvement" in `text-xs text-text-secondary`

**Retraining animation:**
- When `isRetraining` is true, show a progress-like indicator:
  - CSS pulsing animation on the status badge: `animate-pulse`
  - Muted overlay text: "Model retraining in progress…"

**Empty state:**
- If no old/new model data: "No retraining history available" placeholder
