<objective>
Add `beforeAll` API health checks and real-data guards to all 4 Playwright E2E spec files so that tests FAIL (skip) when the backend has no real data, instead of silently passing via mock/empty fallback paths.

Problem: Currently, tests assert on "AAPL"/"MSFT"/"Winner" which appear in BOTH fixture data AND frontend mock fallbacks. If `page.route()` stubs don't fire (URL mismatch, route ordering issue), the frontend mock fills in — and tests pass incorrectly.

Fix: Add a `test.beforeAll` to each `test.describe` block that:
1. Checks `GET http://localhost:8000/health` — skips suite if backend unreachable
2. Checks the relevant data endpoint — skips suite if no data rows returned

This means tests ONLY run against a live, seeded backend. Tests using `page.route()` fixture stubs continue to work as before — but now a live backend is a prerequisite.
</objective>
