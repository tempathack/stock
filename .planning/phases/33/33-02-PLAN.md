Create the ConfigMap following the project's existing format from `k8s/processing/configmap.yaml` and `k8s/ingestion/configmap.yaml`:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: ml-pipeline-config
  namespace: ml
  labels:
    app.kubernetes.io/part-of: stock-prediction-platform
    app.kubernetes.io/component: ml-pipeline
data:
  DATABASE_URL: "postgresql://stockuser:stockpass@postgresql.storage.svc.cluster.local:5432/stockdb"
  MODEL_REGISTRY_DIR: "/data/model_registry"
  SERVING_DIR: "/data/models/active"
  DRIFT_LOG_DIR: "/data/drift_logs"
  LOG_LEVEL: "INFO"
  PIPELINE_VERSION: "1.0.0"
```

Config key rationale:
- **DATABASE_URL**: Full connection string for psycopg2. Uses K8s cross-namespace DNS `postgresql.storage.svc.cluster.local` to reach the PostgreSQL pod in storage namespace. Credentials match `k8s/storage/configmap.yaml` (stockuser) and the Secret (stockpass). Note: Phase 36 (Secrets Management) will move the password to a K8s Secret; for now this matches the existing pattern used by model-serving.yaml.
- **MODEL_REGISTRY_DIR**: Where model artifacts (pipeline.pkl, metadata.json) are stored. `/data/` prefix because Phase 34 mounts a PVC at `/data`.
- **SERVING_DIR**: Where the deployed (active) model is copied for the serving API. Subdirectory of `/data/` on the same PVC.
- **DRIFT_LOG_DIR**: Where drift detection writes event logs (events.jsonl, retraining_log.jsonl).
- **LOG_LEVEL**: Configurable logging level for structured logging.
- **PIPELINE_VERSION**: Matches the `PIPELINE_VERSION = "1.0.0"` constant in `training_pipeline.py` for audit trail consistency.

Usage by Phase 34 CronJobs:
```yaml
envFrom:
  - configMapRef:
      name: ml-pipeline-config
```
