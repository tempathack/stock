<task type="auto">
  <name>Task 2: Create ml/Dockerfile</name>
  <files>ml/Dockerfile</files>
  <action>

Create a multi-stage Dockerfile following the project's established pattern from `services/api/Dockerfile` and `services/kafka-consumer/Dockerfile`:

```dockerfile
# Stage 1 -- Builder
FROM python:3.11-slim AS builder

WORKDIR /app

COPY ml/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Stage 2 -- Runtime
FROM python:3.11-slim

# libpq5: psycopg2 runtime dependency
# libgomp1: OpenMP for XGBoost/LightGBM parallel tree building
RUN apt-get update && \
    apt-get install -y --no-install-recommends libpq5 libgomp1 && \
    rm -rf /var/lib/apt/lists/*

RUN useradd --create-home --uid 1000 appuser

WORKDIR /app

COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin
COPY ml/ ./ml/

USER appuser

ENV PYTHONUNBUFFERED=1
ENV PYTHONDONTWRITEBYTECODE=1

CMD ["python", "-m", "ml.pipelines.training_pipeline"]
```

Key design decisions:
- Build context is project root → `COPY ml/requirements.txt .` and `COPY ml/ ./ml/` work
- No EXPOSE — batch job container, not a server
- No HEALTHCHECK — job completion is tracked by K8s Job status
- libgomp1 required by XGBoost/LightGBM for OpenMP parallelism
- Default CMD is training_pipeline; CronJobs override via command/args for drift_pipeline

Add an `ml-pipeline` service to the existing `docker-compose.yml`. This service uses the project root as build context with `ml/Dockerfile`.

Add after the existing `frontend` service:

```yaml
  ml-pipeline:
    build:
      context: .
      dockerfile: ml/Dockerfile
    image: stock-ml-pipeline:latest
    environment:
      - DATABASE_URL=postgresql://stockuser:stockpass@postgres:5432/stockdb
      - MODEL_REGISTRY_DIR=/data/model_registry
      - SERVING_DIR=/data/models/active
      - DRIFT_LOG_DIR=/data/drift_logs
    depends_on:
      - postgres
```

Note: No `ports` mapping (not a server). Environment variables match the K8s ConfigMap values from Plan 02. `depends_on: postgres` ensures DB is available for data loading.
