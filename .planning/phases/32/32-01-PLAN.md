Refactor the data flow in Forecasts.tsx. Current problems:
1. `allRows` returns full mock when *either* query data is missing — but "missing" includes "still loading"
2. The page effectively always shows mock data on first render because both queries start as undefined

Target pattern for allRows:

```tsx
// BEFORE:
const allRows = useMemo<ForecastRow[]>(() => {
  if (bulkQuery.data && marketQuery.data) {
    return joinForecastData(bulkQuery.data.predictions, marketQuery.data.stocks);
  }
  return generateMockForecasts();
}, [bulkQuery.data, marketQuery.data]);

// AFTER:
const allRows = useMemo<ForecastRow[]>(() => {
  if (bulkQuery.data && marketQuery.data) {
    return joinForecastData(bulkQuery.data.predictions, marketQuery.data.stocks);
  }
  // Only fall back to mock when at least one query has errored
  if (bulkQuery.isError || marketQuery.isError) {
    return generateMockForecasts();
  }
  return [];
}, [bulkQuery.data, marketQuery.data, bulkQuery.isError, marketQuery.isError]);
```

For the StockDetailSection indicator series, same pattern as Dashboard:

```tsx
// BEFORE:
if (indicatorQuery.data?.series && indicatorQuery.data.series.length > 0) {

if (indicatorQuery.data?.series && indicatorQuery.data.series.length > 0) {
  return indicatorQuery.data.series;

if (indicatorQuery.isError) return generateMockIndicatorSeries(ticker);

```

The `isLoading` guard already exists and returns `<LoadingSpinner />` — keep it.
The `isError` guard already exists — keep it.

- The `isError` check should trigger error only when **both** queries fail **and** no cached data — keep existing logic
