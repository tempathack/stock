    # --- services/api/app/services/price_feed.py (NEW) ---
    - Module-level docstring: "Real-time price feed — fetch latest prices and broadcast to WebSocket subscribers."
    - Imports: asyncio, datetime, logging, json, yfinance (yf), pytz
    - Uses US/Eastern timezone for market hours checks

    # --- is_market_open() → bool ---
    - Returns True if current US/Eastern time is Mon–Fri, 09:30–16:00
    - Returns False on weekends and outside market hours
    - Uses pytz for timezone-aware comparison (or zoneinfo if available)

    # --- fetch_live_prices(tickers: list[str]) → list[dict] ---
    - Calls yfinance to get current price snapshot for given tickers
    - Returns list of dicts with keys: ticker, price, change_pct, volume, timestamp
    - Uses yf.download(tickers, period="1d", interval="1m", progress=False) and takes last row
    - Falls back to yf.Ticker(t).info["regularMarketPrice"] if download fails
    - Returns empty list on complete failure (never raises)
    - Logs warning on partial fetch failures

    # --- PriceBroadcaster class ---
    - Manages a set of connected asyncio.Queue instances (one per WebSocket client)
    - subscribe() → asyncio.Queue: creates new queue, adds to subscribers set, returns it
    - unsubscribe(queue): removes queue from subscribers set
    - broadcast(message: dict): puts message dict onto every subscriber queue (non-blocking put_nowait, discards on full queue)
    - Singleton instance: broadcaster = PriceBroadcaster()

    # --- price_feed_loop(tickers: list[str], interval: float = 5.0) → never ---
    - Async coroutine that runs forever
    - Every `interval` seconds: checks is_market_open()
    - If market open: calls fetch_live_prices(tickers) in executor (run_in_executor) to avoid blocking event loop, broadcasts result via broadcaster.broadcast()
    - If market closed: broadcasts {"status": "market_closed", "next_open": <next open ISO string>} once, then sleeps 60s between rechecks
    - On fetch error: logs error, continues loop (never crashes)

    # --- services/api/app/routers/ws.py (NEW) ---
    - Module-level docstring: "WebSocket router — /ws/prices real-time price feed."
    - Imports: asyncio, json, logging, fastapi.APIRouter, fastapi.WebSocket, fastapi.WebSocketDisconnect
    - Imports broadcaster from app.services.price_feed

    # --- router = APIRouter(tags=["websocket"]) ---

    # --- @router.websocket("/ws/prices") ---
    - Accepts WebSocket connection via await websocket.accept()
    - Subscribes to broadcaster via queue = broadcaster.subscribe()
    - Enters try/finally loop:
      - try: awaits queue.get(), sends JSON text via websocket.send_json(message)
      - Concurrently listens for client messages (ping/pong or close) using asyncio.create_task for receive
      - On WebSocketDisconnect: breaks loop
      - finally: broadcaster.unsubscribe(queue), logs disconnect
    - Logs connection/disconnection with client host info

<objective>
Add a WebSocket endpoint at /ws/prices that pushes live stock price updates to all
connected clients every 5 seconds during US market hours. The endpoint uses a
publish-subscribe broadcaster pattern so multiple frontend clients can connect
simultaneously. During market-closed hours, clients receive a status message and the
feed pauses to conserve API calls.

Purpose: FENH-01 (WebSocket endpoint /ws/prices pushing price updates every 5s during market hours).
Output: New WebSocket router, price feed service with market-hours awareness, and
background task lifecycle management in the FastAPI lifespan.
</objective>
