    # --- services/frontend/src/hooks/useWebSocket.ts (NEW) ---
    - Module exports a custom React hook: useWebSocket(url: string, options?: UseWebSocketOptions)
    - UseWebSocketOptions interface:
      - enabled?: boolean (default true — allows conditional connection)
      - reconnectAttempts?: number (default 5)
      - reconnectInterval?: number (default 3000ms)
      - onMessage?: (data: WebSocketMessage) => void (optional callback)
    - Return type: { status: WebSocketStatus; lastMessage: WebSocketMessage | null; prices: Map<string, WebSocketPriceUpdate> }

    # --- Hook internal behavior ---
    - Uses useRef for WebSocket instance (prevents stale closure issues)
    - Uses useRef for reconnect attempt counter
    - Uses useRef for reconnect timeout ID (for cleanup)
    - Uses useState for status (WebSocketStatus)
    - Uses useState for lastMessage (WebSocketMessage | null)
    - Uses useRef with useState trigger for prices Map (avoids re-render on every Map mutation — only triggers re-render on new message batch)

    # --- Connection lifecycle ---
    - useEffect with [url, enabled] dependencies:
      - If !enabled or !url: close existing connection, return
      - Creates new WebSocket(url)
      - onopen: set status "connected", reset reconnect counter to 0
      - onmessage: parse JSON, update lastMessage state, update prices Map (for price_update type: iterate prices array, set each ticker in Map), call options.onMessage if provided
      - onerror: set status "error", log to console.error
      - onclose: set status "disconnected", attempt reconnect if counter < reconnectAttempts
      - Cleanup function: close WebSocket (code 1000), clear reconnect timeout
    - Reconnect logic: incrementing delay using reconnectInterval * attempt (linear backoff, capped at reconnectAttempts)

    # --- Reconnect behavior ---
    - On unexpected close (not code 1000): schedule reconnect via setTimeout
    - Each reconnect attempt: increment counter, set status "connecting", create new WebSocket
    - After max attempts: stay "disconnected", log warning "Max reconnect attempts reached"
    - On successful reconnect: reset counter to 0
