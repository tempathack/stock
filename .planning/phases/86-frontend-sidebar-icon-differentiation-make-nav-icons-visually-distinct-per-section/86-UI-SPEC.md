## Problem Statement

The six nav icons in `TopNav` (`Sidebar.tsx`) are visually indistinct. Four of them (`AccountTreeIcon`, `BubbleChartIcon`, `SsidChartIcon`, `BarChartIcon`) share a similar thin-line chart motif. A user scanning the nav bar cannot identify a section by its icon silhouette alone.

**Goal:** Replace icons so each section has a unique, semantically appropriate silhouette at 15px rendered size within the active/inactive color contract.

---

## Icon Assignment Contract

Each icon must have a distinct silhouette. No two icons may share the same visual family (e.g., two "chart-style" icons).

| Nav Section | Route | Current Icon | Replacement Icon | Semantic Rationale |
|-------------|-------|-------------|------------------|--------------------|
| Dashboard | `/dashboard` | `DashboardIcon` | `DashboardIcon` (keep) | Grid/tile silhouette is unique and accurate |
| Models | `/models` | `AccountTreeIcon` | `PsychologyIcon` | Brain/neural network — distinct blob shape, represents ML models |
| Forecasts | `/forecasts` | `TrendingUpIcon` | `TrendingUpIcon` (keep) | Arrow-up silhouette is maximally distinct for predictions |
| Drift | `/drift` | `BubbleChartIcon` | `WaterDropIcon` | Droplet silhouette — represents distribution drift, unique shape |
| Backtest | `/backtest` | `SsidChartIcon` | `HistoryIcon` | Clock+arrow silhouette — represents historical replay |
| Analytics | `/analytics` | `BarChartIcon` | `InsightsIcon` | Sparkline+dot silhouette — distinct from bar chart, represents analytics |

**Import paths** (all from `@mui/icons-material`):
```
import PsychologyIcon from "@mui/icons-material/Psychology";
import WaterDropIcon  from "@mui/icons-material/WaterDrop";
import HistoryIcon    from "@mui/icons-material/History";
import InsightsIcon   from "@mui/icons-material/Insights";
```

**Fallback note:** If `PsychologyIcon` is not available in the installed version, use `AutoAwesomeIcon`. If `WaterDropIcon` is unavailable, use `WavesIcon`. Check availability via `import` before shipping.

## Spacing Scale

Declared values (must be multiples of 4) — source: existing codebase patterns in `Sidebar.tsx`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap within nav item (`gap: 0.75` MUI spacing = 6px, nearest 4px step is 8px; icon internal padding) |
| sm | 8px | Compact element spacing, icon-to-label gap |
| md | 16px | Default element spacing, nav item horizontal padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Nav items use `px: 1.5` (12px) MUI spacing for horizontal padding — intentional compact fit within 60px header height. Touch targets are 60px tall (full header height), exceeding 44px minimum. No additional exceptions.

## Color

Source: `globals.css` `@theme` block.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#02000A` | Page background, body surface |
| Secondary (30%) | `#07041A` / `#0D0A24` | Header background (`rgba(7,4,26,0.85)`), cards |
| Accent (10%) | `#7C3AED` (purple) + `#00F5FF` (cyan) | Active nav indicator underline, active icon glow |
| Destructive | `#FF2D78` | Error states, loss indicators — not used in this phase |

Accent reserved for: active nav item underline gradient (`#7C3AED → #00F5FF`), active icon color (`#BF5AF2`), active icon drop-shadow filter.

**Icon color contract (source: Sidebar.tsx lines 231–238):**
- Inactive icon: inherits nav item color `#4A4270`, opacity 0.5
- Active icon: `#BF5AF2` (neon purple), opacity 1.0, `drop-shadow(0 0 4px rgba(191,90,242,0.7))`
- Hover (inactive): nav item color transitions to `#9D8FD8`

This color contract applies identically to replacement icons — no color changes in this phase.

## Interaction Contract

This phase is a pure icon swap. No new interaction states are introduced. The existing interaction model is preserved exactly:

| State | Icon opacity | Icon color | Nav item color | Underline |
|-------|-------------|------------|---------------|-----------|
| Inactive | 0.5 | inherits `#4A4270` | `#4A4270` | none |
| Active | 1.0 | `#BF5AF2` | `#F0EEFF` | `linear-gradient(90deg, #7C3AED, #00F5FF)`, 2px, 60% width |
| Hover (inactive) | 0.5 | `#9D8FD8` | `#9D8FD8` | none |
| Hover (active) | 1.0 | `#BF5AF2` | `#F0EEFF` | unchanged |

Icon rendered size: `fontSize: "0.95rem"` (approx 15px at 16px base). This value is not changed.

## Accessibility Contract

- Each icon must have an implicit accessible name via the visible text label that follows it in the DOM (existing pattern).
- Icons are `aria-hidden="true"` by default in MUI — this is correct because the adjacent `Typography` label provides the accessible name.
- No `aria-label` changes required.
- Icon silhouettes must remain recognizable at 15px. All four replacement icons (`PsychologyIcon`, `WaterDropIcon`, `HistoryIcon`, `InsightsIcon`) have distinct silhouettes that are legible at this size.

| Icon | Visual Family |
