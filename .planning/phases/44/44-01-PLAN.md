    - Module-level docstring: "Feature store — precompute, persist, and read feature vectors from PostgreSQL."

    - Imports DBSettings from ml.pipelines.components.data_loader

    # --- compute_features_for_ticker(ticker, conn) → pd.DataFrame ---
    - Loads OHLCV data via load_ticker_data(ticker, conn)
    - Applies compute_all_indicators, compute_lag_features, compute_rolling_stats in sequence
    - Calls drop_incomplete_rows to remove NaN warm-up rows
    - Returns enriched DataFrame (does NOT include target column — targets are training-time concern)
    - Logs info: "Computed {n_features} features for {ticker} ({n_rows} rows)"
    - Returns empty DataFrame if no OHLCV data available (logs warning)

    # --- write_features(ticker, df, conn) → int ---
    - Takes ticker string, enriched DataFrame (DatetimeIndex), and psycopg2 connection
    - Iterates over rows and feature columns, building list of (ticker, date, feature_name, feature_value) tuples
    - Uses executemany with INSERT ... ON CONFLICT (ticker, date, feature_name) DO UPDATE SET feature_value = EXCLUDED.feature_value, computed_at = NOW()
    - Commits after batch insert
    - Returns count of rows upserted
    - Logs info: "Wrote {count} feature rows for {ticker}"
    - Batches inserts in chunks of 5000 to avoid memory issues with large tickers

    # --- compute_and_store(tickers, settings) → dict[str, int] ---
    - Top-level orchestrator: connects to DB, iterates tickers, calls compute_features_for_ticker + write_features
    - Returns dict mapping ticker → row count written
    - Connection opened once and shared across all tickers (closed in finally block)
    - Logs summary: "Feature store updated: {n} tickers, {total} rows"
    - If a single ticker fails, logs error and continues to next ticker (no abort)

    # --- read_features(tickers, settings, start_date, end_date) → dict[str, pd.DataFrame] ---
    - Reads precomputed features from feature_store table
    - Query: SELECT date, feature_name, feature_value FROM feature_store WHERE ticker = %s [AND date >= %s] [AND date <= %s] ORDER BY date
    - Pivots result from EAV to wide DataFrame: rows=dates, columns=feature_names
    - Returns dict[str, pd.DataFrame] matching the signature of engineer_features output
    - Returns empty dict if no data found
    - Logs info per ticker: "Read {n_rows} × {n_features} from feature store for {ticker}"

    # --- get_feature_freshness(ticker, conn) → date | None ---
    - Returns the max(date) in feature_store for a given ticker
    - Used by CronJob to determine if incremental update is needed
    - Returns None if no features exist for the ticker

<objective>
Create the feature store infrastructure: a PostgreSQL table for precomputed features
and a Python module to compute features from OHLCV data, write them to PostgreSQL,
and read them back as wide DataFrames. This decouples feature computation from model
training, enabling faster training runs and shared feature access across services.

Purpose: ADVML-07 (feature_store PostgreSQL table with precomputed features).
Output: New database table, Python read/write module, ORM model, and full test coverage.
The table uses EAV format (ticker, date, feature_name, feature_value) with upsert
semantics for idempotent daily updates. The read_features function pivots EAV rows
back into wide DataFrames matching the format expected by the training pipeline.
</objective>
