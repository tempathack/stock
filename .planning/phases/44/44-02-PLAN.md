<objective>
Wire the feature store into production: a daily K8s CronJob that precomputes all features
after market close, a CLI entry point that the CronJob invokes, and training pipeline
integration so it reads from the feature store instead of computing on-the-fly.

Purpose: ADVML-07 (daily CronJob) + ADVML-08 (training pipeline reads from feature store).
Output: The CronJob runs weekdays at 6 PM UTC (after US market close), computing all
indicators/lags/rolling stats for every active ticker and upserting to the feature_store
table. The training pipeline's engineer_features step gains a use_feature_store toggle:
when enabled, it reads precomputed features from PostgreSQL, falling back to on-the-fly
computation for any tickers missing from the store. This makes training ~10x faster for
the feature engineering step and ensures all services consume identical feature values.
</objective>
