  <files>k8s/monitoring/promtail-configmap.yaml</files>
  <action>

Create the Promtail configuration ConfigMap:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: promtail-config
  namespace: monitoring
data:
  promtail.yaml: |
    server:
      http_listen_port: 9080
      grpc_listen_port: 0

    positions:
      filename: /run/promtail/positions.yaml

    clients:
      - url: http://loki.monitoring.svc.cluster.local:3100/loki/api/v1/push

    scrape_configs:
      - job_name: kubernetes-pods
        kubernetes_sd_configs:
          - role: pod
        relabel_configs:
          # Keep only pods that are running
          - source_labels: [__meta_kubernetes_pod_phase]
            action: keep
            regex: Running

          # Use pod name as __address__ 
          - source_labels: [__meta_kubernetes_pod_node_name]
            target_label: __host__

          # Extract namespace label
          - source_labels: [__meta_kubernetes_namespace]
            target_label: namespace

          # Extract pod name label
          - source_labels: [__meta_kubernetes_pod_name]
            target_label: pod

          # Extract container name label
          - source_labels: [__meta_kubernetes_pod_container_name]
            target_label: container

          # Build the log file path
          - source_labels:
              - __meta_kubernetes_namespace
              - __meta_kubernetes_pod_name
              - __meta_kubernetes_pod_container_name
            separator: /
            target_label: __path__
            replacement: /var/log/pods/*$1/$2/$3/*.log

        pipeline_stages:
          - cri: {}
```

Key points:
- Uses `kubernetes_sd_configs` with `role: pod` for automatic discovery
- Relabeling adds namespace, pod, container labels to every log stream
- CRI pipeline stage strips the container runtime log wrapper (timestamp + stream + flags)
- `__path__` construction matches K8s standard pod log file locations

  <name>Task 6: Create k8s/monitoring/promtail-daemonset.yaml</name>
  <files>k8s/monitoring/promtail-daemonset.yaml</files>
  <action>

Create the Promtail DaemonSet:

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: promtail
  namespace: monitoring
  labels:
    app: promtail
spec:
  selector:
    matchLabels:
      app: promtail
  template:
    metadata:
      labels:
        app: promtail
    spec:
      serviceAccountName: promtail
      tolerations:
        - key: node-role.kubernetes.io/master
          effect: NoSchedule
        - key: node-role.kubernetes.io/control-plane
          effect: NoSchedule
      containers:
        - name: promtail
          image: grafana/promtail:2.9.6
          args:
            - -config.file=/etc/promtail/promtail.yaml
          ports:
            - containerPort: 9080
              name: http-metrics
          resources:
            requests:
              cpu: 50m
              memory: 64Mi
            limits:
              cpu: 100m
              memory: 128Mi
          volumeMounts:
            - name: promtail-config
              mountPath: /etc/promtail
            - name: varlogpods
              mountPath: /var/log/pods
              readOnly: true
            - name: varlibdockercontainers
              mountPath: /var/lib/docker/containers
              readOnly: true
            - name: run
              mountPath: /run/promtail
      volumes:
        - name: promtail-config
          configMap:
            name: promtail-config
        - name: varlogpods
          hostPath:
            path: /var/log/pods
        - name: varlibdockercontainers
          hostPath:
            path: /var/lib/docker/containers
        - name: run
          emptyDir: {}
```

Key points:
- DaemonSet ensures one Promtail pod per node (single node in Minikube)
- hostPath mounts for /var/log/pods and /var/lib/docker/containers (readOnly)
- Tolerations for master/control-plane nodes so Promtail can collect system pod logs
- emptyDir for positions file (tracks read offsets across restarts)
