  <name>Task 1: Create services/api/entrypoint.sh</name>
  <files>services/api/entrypoint.sh</files>
  <action>

Create the entrypoint script at `services/api/entrypoint.sh`:

```bash
#!/bin/bash
set -e

# Run Alembic migrations if DATABASE_URL is configured
if [ -n "$DATABASE_URL" ]; then
    echo "[entrypoint] Running database migrations..."
    alembic upgrade head
    echo "[entrypoint] Migrations complete."
else
    echo "[entrypoint] DATABASE_URL not set — skipping migrations."
fi

# Start the API server (exec replaces shell process for proper signal handling)
echo "[entrypoint] Starting uvicorn..."
exec uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Key points:
- `set -e` ensures migration failures prevent uvicorn from starting (fail-fast)
- Conditional on DATABASE_URL so the container can still start without a DB (Phase 3 compat)
- `exec` makes uvicorn PID 1 in the container for proper SIGTERM handling and K8s graceful shutdown
- Log messages provide observability in `kubectl logs`

Update the Dockerfile to include Alembic configuration files and use the entrypoint script.

**Changes:**
1. After `COPY ./app ./app`, add:
   ```dockerfile
   COPY alembic.ini .
   COPY ./alembic ./alembic
   COPY entrypoint.sh .
   ```
2. Before `USER appuser`, add:
   ```dockerfile
   RUN chmod +x entrypoint.sh
   ```
3. Replace the final `CMD` line:
   ```dockerfile
   # Old:
   CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
   
   # New:
   CMD ["./entrypoint.sh"]
   ```

The resulting Dockerfile should be:

```dockerfile
# Stage 1 -- Builder
FROM python:3.11-slim AS builder

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Stage 2 -- Runtime
FROM python:3.11-slim

RUN apt-get update && apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*
RUN useradd --create-home --uid 1000 appuser

WORKDIR /app

COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin
COPY ./app ./app
COPY alembic.ini .
COPY ./alembic ./alembic
COPY entrypoint.sh .
RUN chmod +x entrypoint.sh

USER appuser

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=10s --start-period=10s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

CMD ["./entrypoint.sh"]
```

Key points:
- Alembic files MUST be copied before `USER appuser` so they're owned by root (read-only for appuser is fine)
- `chmod +x` before USER switch ensures entrypoint is executable
- HEALTHCHECK unchanged — still validates uvicorn is running
- Start period of 10s gives time for migration + uvicorn start
