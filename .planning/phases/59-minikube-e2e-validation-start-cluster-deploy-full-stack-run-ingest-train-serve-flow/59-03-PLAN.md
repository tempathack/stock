  <name>Task 1: Seed DB and trigger historical ingestion → training pipeline</name>
  <files></files>
  <read_first>
    - stock-prediction-platform/scripts/seed-data.sh
  </read_first>
  <action>
    Run from /home/tempa/Desktop/priv-project/stock-prediction-platform:

    STEP A — Seed the database using the seed script (faster and more reliable than full ingestion):
    ```bash
    cd /home/tempa/Desktop/priv-project/stock-prediction-platform

    # Port-forward PostgreSQL so seed script can reach it from host
    kubectl port-forward svc/postgresql -n storage 5432:5432 &
    PF_PID=$!
    sleep 3

    bash scripts/seed-data.sh
    kill $PF_PID 2>/dev/null || true
    ```

    If seed-data.sh requires environment variables, set them first:
    ```bash
    export PGPASSWORD=devpassword123
    export PGHOST=localhost
    export PGPORT=5432
    export PGUSER=stockuser
    export PGDATABASE=stockdb
    ```

    After seeding, verify row count:
    ```bash
    PG_POD=$(kubectl get pod -n storage -l app=postgresql -o jsonpath='{.items[0].metadata.name}')
    kubectl exec -n storage "$PG_POD" -- psql -U stockuser -d stockdb -c "SELECT COUNT(*) FROM ohlcv_daily;"
    ```
    Expected: COUNT >= 100 rows.

    STEP B — Also trigger the historical ingestion CronJob to get real Yahoo Finance data
    (runs in parallel with seed; the training job will use whichever data lands in DB first):
    ```bash
    kubectl create job --from=cronjob/historical-ingestion manual-ingest-59 -n ingestion \
      --dry-run=client -o yaml | kubectl apply -f - 2>/dev/null || \
    kubectl create job --from=cronjob/historical-ingestion manual-ingest-59 -n ingestion
    kubectl wait --for=condition=complete job/manual-ingest-59 -n ingestion --timeout=600s
    kubectl logs job/manual-ingest-59 -n ingestion
    ```

    If ingestion job fails (Yahoo Finance unreachable or rate limited), proceed with seed data only —
    the seed data is sufficient for training.

    STEP C — Trigger the weekly training CronJob:
    ```bash
    kubectl create job --from=cronjob/weekly-training manual-train-59 -n ml \
      --dry-run=client -o yaml | kubectl apply -f - 2>/dev/null || \
    kubectl create job --from=cronjob/weekly-training manual-train-59 -n ml
    echo "Training job started — waiting up to 10 minutes..."
    kubectl wait --for=condition=complete job/manual-train-59 -n ml --timeout=600s
    kubectl logs job/manual-train-59 -n ml | tail -50
    ```

    If training job fails, check logs:
    ```bash
    kubectl describe job manual-train-59 -n ml
    kubectl logs job/manual-train-59 -n ml
    ```
    Common fix: ensure stock-platform-secrets and minio-secrets are in the ml namespace
    (deploy-all.sh Phase 36 and 51 blocks copy them — verify with kubectl get secret -n ml).
  </action>

    Run from /home/tempa/Desktop/priv-project/stock-prediction-platform:

    STEP A — Verify model artifact in MinIO:
    ```bash
    MINIO_POD=$(kubectl get pod -n storage -l app=minio -o jsonpath='{.items[0].metadata.name}')

    # Try mc ls (available in MinIO image)
    kubectl exec -n storage "$MINIO_POD" -- mc ls local/model-artifacts/serving/active/ 2>/dev/null \
      || kubectl exec -n storage "$MINIO_POD" -- ls /data/model-artifacts/serving/active/ 2>/dev/null \
      || {
        # Fallback: port-forward and use curl S3 API
        kubectl port-forward svc/minio -n storage 9000:9000 &
        MINIO_PF=$!
        sleep 3
        curl -s "http://minioadmin:minioadmin123@localhost:9000/model-artifacts/serving/active/" | grep -E "model.joblib|model-settings.json"
        kill $MINIO_PF
      }
    ```

    Both `model.joblib` (or `model.pkl`) and `model-settings.json` must appear.
    If only one file exists, the training pipeline may not have written model-settings.json —
    check if that file is needed by the serving runtime and create it manually if missing:
    ```bash
    # model-settings.json content for MLServer sklearn runtime:
    # {"name": "stock-model-serving", "implementation": "mlserver_sklearn.SKLearnModel"}
    # Inject via kubectl exec into MinIO if missing
    ```

    STEP B — Force KServe InferenceService to reconcile:
    ```bash
    # Restart predictor deployment to pick up the newly available artifact
    kubectl rollout restart deployment -n ml \
      -l serving.kserve.io/inferenceservice=stock-model-serving 2>/dev/null || true

    # Wait for InferenceService Ready (up to 10 minutes — model download from MinIO takes time)
    kubectl wait --for=condition=Ready inferenceservice/stock-model-serving -n ml --timeout=600s
    kubectl get inferenceservice -n ml
    kubectl get pods -n ml -l serving.kserve.io/inferenceservice=stock-model-serving
    ```

    If the predictor pod is in CrashLoopBackOff, check logs:
    ```bash
    PRED_POD=$(kubectl get pod -n ml -l serving.kserve.io/inferenceservice=stock-model-serving -o jsonpath='{.items[0].metadata.name}')
    kubectl logs -n ml "$PRED_POD" --previous 2>/dev/null || kubectl logs -n ml "$PRED_POD"
    ```

    Common issue: MLServer cannot find model-settings.json — ensure it is present in MinIO.

    STEP C — Validate /predict/AAPL returns a prediction:
    ```bash
    API_POD=$(kubectl get pod -n ingestion -l app=stock-api -o jsonpath='{.items[0].metadata.name}')
    kubectl exec -n ingestion "$API_POD" -- \
      curl -s http://localhost:8000/predict/AAPL | python3 -m json.tool
    ```

    Expected response shape (from kserve_client.py):
    ```json
    {
      "predicted_price": 185.42,
      "ticker": "AAPL",
      "model_name": "...",
      "horizon": 1
    }
    ```

    The key requirement: `predicted_price` is a positive float (not null, not 0).

    STEP D — Run integration tests:
    ```bash
    cd /home/tempa/Desktop/priv-project/stock-prediction-platform
    ```
