# Plan 08-01: K8s CronJob Manifests, Trigger Scripts, and Deployment Wiring

## Objective

Deliver two K8s CronJob manifests (`cronjob-intraday.yaml` and `cronjob-historical.yaml`)
in the `ingestion` namespace that call the FastAPI `/ingest/intraday` and `/ingest/historical`
endpoints on schedule. Provide Python trigger scripts invoked by the CronJob containers,
wire them into `deploy-all.sh`, and validate with unit tests.

Satisfies: INGEST-04 (intraday CronJob, market hours) and INGEST-05 (historical CronJob, weekly).

---

## must_haves

- `cronjob-intraday.yaml` exists at `k8s/ingestion/cronjob-intraday.yaml` with schedule
  `*/5 14-21 * * 1-5`, namespace `ingestion`, `concurrencyPolicy: Forbid`, and
  `restartPolicy: OnFailure`
- `cronjob-historical.yaml` exists at `k8s/ingestion/cronjob-historical.yaml` with schedule
  `0 2 * * 0`, namespace `ingestion`, `concurrencyPolicy: Forbid`, and
  `restartPolicy: OnFailure`
- Both CronJobs reference `configMapRef: ingestion-config` for environment injection
- Both CronJobs use `image: stock-api:latest` with `imagePullPolicy: Never` (local Minikube)
- `trigger_intraday.py` invoked as `python -m app.jobs.trigger_intraday`, POSTs to
  `{API_BASE_URL}/ingest/intraday`, exits 0 on HTTP 200, exits 1 on any error
- `trigger_historical.py` invoked as `python -m app.jobs.trigger_historical`, POSTs to
  `{API_BASE_URL}/ingest/historical`, exits 0 on HTTP 200, exits 1 on any error
- `httpx` is in `services/api/requirements.txt`
- `deploy-all.sh` applies both CronJob YAMLs under a `[Phase 8]` echo block
- Unit tests in `tests/test_cronjob_triggers.py` cover success and error paths for both
  trigger modules

```python
"""CronJob trigger for intraday ingestion.

Invoked by K8s CronJob via: ``python -m app.jobs.trigger_intraday``

Makes an HTTP POST to the running FastAPI service at /ingest/intraday
and exits with code 0 on success, 1 on failure.
"""

from __future__ import annotations

import os
import sys

import httpx

from app.utils.logging import get_logger

logger = get_logger(__name__)

API_BASE_URL = os.environ.get(
    "API_BASE_URL", "http://stock-api.ingestion.svc.cluster.local:8000"
)
REQUEST_TIMEOUT = int(os.environ.get("TRIGGER_TIMEOUT_SECONDS", "300"))


def main() -> int:
    """POST to /ingest/intraday and return exit code."""
    url = f"{API_BASE_URL}/ingest/intraday"
    logger.info("trigger_intraday_start", url=url, timeout=REQUEST_TIMEOUT)

    try:
        response = httpx.post(url, timeout=REQUEST_TIMEOUT)
        response.raise_for_status()
    except httpx.HTTPStatusError as exc:
        logger.error(
            "trigger_intraday_http_error",
            status_code=exc.response.status_code,
            detail=exc.response.text,
        )
        return 1
    except httpx.RequestError as exc:
        logger.error("trigger_intraday_request_error", error=str(exc))
        return 1

    data = response.json()
    logger.info(
        "trigger_intraday_complete",
        status=data.get("status"),
        records_fetched=data.get("records_fetched"),
        records_produced=data.get("records_produced"),
    )
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

The module reads `API_BASE_URL` and `TRIGGER_TIMEOUT_SECONDS` from environment (injected by
the CronJob's `envFrom: configMapRef: ingestion-config`). It uses `httpx.HTTPStatusError`
and `httpx.RequestError` as the two distinct failure modes.
</action>
