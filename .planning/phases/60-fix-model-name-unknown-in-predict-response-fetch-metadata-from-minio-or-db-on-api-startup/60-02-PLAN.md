**Step 2 — Update `stock-prediction-platform/k8s/ingestion/fastapi-deployment.yaml`:**

In the `spec.template.spec.containers[0].envFrom` block, add a `secretRef` for `minio-secrets` AFTER the existing configMapRef:
```yaml
          envFrom:
            - configMapRef:
                name: ingestion-config
            - secretRef:
                name: minio-secrets
```

This exposes `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` from the Secret to the API Pod environment.

**Step 3 — Check if `minio-secrets` exists in `ingestion` namespace:**

Run: `kubectl get secret minio-secrets -n ingestion 2>&1`

If the secret does NOT exist in `ingestion` namespace, copy it from `storage` using:
```bash
MINIO_USER=$(kubectl get secret minio-secrets -n storage -o jsonpath='{.data.MINIO_ROOT_USER}')
MINIO_PASS=$(kubectl get secret minio-secrets -n storage -o jsonpath='{.data.MINIO_ROOT_PASSWORD}')
kubectl create secret generic minio-secrets -n ingestion \
  --from-literal=MINIO_ROOT_USER=$(echo "$MINIO_USER" | base64 -d) \
  --from-literal=MINIO_ROOT_PASSWORD=$(echo "$MINIO_PASS" | base64 -d) \
  --dry-run=client -o yaml | kubectl apply -f -
```

**Step 4 — Apply the updated ConfigMap and Deployment:**
```bash
kubectl apply -f stock-prediction-platform/k8s/ingestion/configmap.yaml
kubectl apply -f stock-prediction-platform/k8s/ingestion/fastapi-deployment.yaml
kubectl rollout restart deployment/stock-api -n ingestion
kubectl rollout status deployment/stock-api -n ingestion --timeout=120s
```

**Step 5 — Check API startup logs for metadata load:**
```bash
kubectl logs -n ingestion deployment/stock-api --since=60s | grep -i "model metadata\|minio\|serving_config"
```
