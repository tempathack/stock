## Design Decisions

1. **`trigger_retraining()` function:** The primary programmatic entry point for drift-triggered retraining. Calls `run_training_pipeline()` with explicit parameters. Returns `PipelineRunResult`. This function is what Phase 22's drift auto-retrain will invoke.

   ```python
   def trigger_retraining(
       tickers: list[str] | None = None,
       db_settings: DBSettings | None = None,
       data_dict: dict[str, pd.DataFrame] | None = None,
       registry_dir: str = "model_registry",
       serving_dir: str = "/models/active",
       reason: str = "manual",
       skip_shap: bool = False,
   ) -> PipelineRunResult:
   ```

   The `reason` parameter documents why the pipeline was triggered (e.g., `"manual"`, `"data_drift"`, `"prediction_drift"`, `"concept_drift"`, `"scheduled"`). It is logged and stored alongside the run result.

2. **`submit_pipeline_run()` function:** Submits a compiled KFP pipeline to the Kubeflow Pipelines API. Guarded behind `kfp` import check — raises `ImportError` with helpful message if KFP SDK is not installed. Not unit-tested (requires live KFP infra).

   ```python
   def submit_pipeline_run(
       pipeline_package_path: str,
       host: str = "http://ml-pipeline.ml.svc.cluster.local:8888",
       experiment_name: str = "stock-prediction",
       run_name: str | None = None,
   ) -> str:
   ```

3. **KFP DSL pipeline definition: `compile_kfp_pipeline()`:** Compiles the pipeline as a KFP v2 YAML. Uses `@dsl.pipeline` decorator on a wrapper function. Each of the 11 steps is represented as a `@dsl.component` that reads/writes Parquet artifacts via `dsl.Input[dsl.Dataset]` / `dsl.Output[dsl.Dataset]`. Gated behind `kfp` import availability.

   ```python
   def compile_kfp_pipeline(output_path: str = "training_pipeline.yaml") -> str:
   ```

4. **Test strategy:** Tests for `trigger_retraining()` use the same `data_dict` bypass as Plan 20-01 tests. KFP compilation test is conditional on `kfp` package availability (skip if not installed). No live KFP deployment required for any tests.

5. **Retraining log:** `trigger_retraining()` appends a record to `{registry_dir}/runs/retraining_log.json` (JSON lines format) with `reason`, `run_id`, `timestamp`, and `status`. This provides a queryable history of all retraining triggers for the /drift frontend page.

      ```python
      def trigger_retraining(
          tickers: list[str] | None = None,
          db_settings: DBSettings | None = None,
          data_dict: dict[str, pd.DataFrame] | None = None,
          registry_dir: str = "model_registry",

          skip_shap: bool = False,
      ) -> PipelineRunResult:
          """Trigger a full training pipeline run.

          This is the entry point for both manual and drift-triggered retraining.
          Phase 22 will wire drift detection output to this function.

          Parameters
          ----------
          reason:
              Why the pipeline was triggered. One of: ``"manual"``, ``"data_drift"``,
              ``"prediction_drift"``, ``"concept_drift"``, ``"scheduled"``.
          """
          logger.info("Triggering retraining — reason: %s", reason)

          result = run_training_pipeline(
              tickers=tickers,
              db_settings=db_settings,
              data_dict=data_dict,
              registry_dir=registry_dir,
              serving_dir=serving_dir,
              skip_shap=skip_shap,
          )

          # Append to retraining log
          _append_retraining_log(result, reason, registry_dir)

          logger.info(
              "Retraining %s — run_id=%s, reason=%s",
              result.status, result.run_id, reason,
          )
          return result
      ```

      **`_append_retraining_log()` helper:**
      Appends a JSON record to `{registry_dir}/runs/retraining_log.jsonl`:
      ```json
      {"run_id": "...", "reason": "manual", "timestamp": "...", "status": "completed", "pipeline_version": "1.0.0", "winner": "ridge_standard"}
      ```

      **`compile_kfp_pipeline()` function:**
      Gated behind `try: import kfp; except ImportError`.
      If `kfp` is available:
      - Defines `@dsl.pipeline(name="stock-training-pipeline", description=..., pipeline_root="gs://...")` wrapper
      - Each of the 11 steps uses `@dsl.component(base_image="python:3.11", packages_to_install=[...])` with Parquet I/O
      - Compiles via `kfp.compiler.Compiler().compile(pipeline_func, output_path)`
      - Returns path to compiled YAML
      If `kfp` is not available:
      - Logs warning and raises `ImportError` with install instructions

      **`submit_pipeline_run()` function:**
      Gated behind `kfp` import.
      - Creates `kfp.Client(host=host)`
      - Creates/gets experiment by name
      - Submits pipeline run
      - Returns run ID string

      ```python
      if __name__ == "__main__":
          import argparse
          parser = argparse.ArgumentParser(description="Trigger stock prediction retraining")
          parser.add_argument("--tickers", type=str, help="Comma-separated ticker list")
          parser.add_argument("--reason", default="manual", choices=["manual", "data_drift", "prediction_drift", "concept_drift", "scheduled"])
          parser.add_argument("--registry-dir", default="model_registry")
          parser.add_argument("--serving-dir", default="/models/active")
          parser.add_argument("--compile-only", action="store_true", help="Compile KFP pipeline YAML only")
          parser.add_argument("--skip-shap", action="store_true")
          args = parser.parse_args()
          if args.compile_only:
              path = compile_kfp_pipeline()
              print(f"Compiled pipeline to {path}")
          else:
              tickers = args.tickers.split(",") if args.tickers else None
              result = trigger_retraining(
                  tickers=tickers, reason=args.reason,
                  registry_dir=args.registry_dir, serving_dir=args.serving_dir,
                  skip_shap=args.skip_shap,
              )
              print(json.dumps(result.to_dict(), indent=2, default=str))
      ```
