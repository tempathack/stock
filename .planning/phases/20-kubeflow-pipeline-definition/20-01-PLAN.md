# Plan 20-01 — Parquet Serialisation + Training Pipeline Orchestrator + Versioning

## Objective
Implement the full 11-step training pipeline orchestrator (`training_pipeline.py`), Parquet serialisation helpers (`serialization.py`), and pipeline run versioning. The orchestrator chains all Phase 17–19 components into a single `run_training_pipeline()` function that can be called locally, from a drift trigger, or wrapped in a KFP DSL pipeline. Satisfies **KF-13** (full pipeline definition) and **KF-14** (pipeline versioning).

## Design Decisions

1. **`PipelineRunResult` dataclass:** Captures full audit trail of a pipeline execution:
   ```python
   @dataclass
   class PipelineRunResult:
       run_id: str                    # UUID4
       pipeline_version: str          # "1.0.0"
       started_at: str                # ISO timestamp
       completed_at: str              # ISO timestamp
       status: str                    # "completed" | "failed"
       steps_completed: list[str]     # ["load_data", "engineer_features", ...]
       n_tickers: int                 # number of tickers processed
       n_models_trained: int          # total models trained
       winner_info: dict | None       # from select_and_persist_winner()
       deploy_info: dict | None       # from deploy_winner_model()
       error: str | None              # traceback string if failed
   ```

2. **`run_training_pipeline()` function:** Single entry point that accepts either `tickers` + `db_settings` (production) or `data_dict` (test/pre-loaded). Executes all 11 steps sequentially, records step completion, and saves the run result to the registry directory.

3. **Parquet serialisation:** `save_dataframes()` writes each ticker's DataFrame as `{ticker}.parquet` within a directory. `load_dataframes()` reads them back. Uses `pyarrow` engine (already installed from Phase 13 CatBoost upgrade). Feature names and metadata serialised as JSON sidecar files.

4. **Pipeline version:** Hardcoded `PIPELINE_VERSION = "1.0.0"` constant in `training_pipeline.py`. Incremented manually when pipeline structure changes. Recorded in every `PipelineRunResult` and in model metadata.

5. **Run result persistence:** `pipeline_run_{run_id}.json` saved to `{registry_dir}/runs/` directory. Provides queryable audit trail.

6. **Training data flow:** The orchestrator calls the individual step functions rather than `train_all_models_pipeline()` so that each step is tracked independently. Specifically:
   - Steps 1–3: `load_data` → `engineer_features` → `generate_labels` → `(data_dict, features)`
   - Step 4: `prepare_training_data(data_dict, features)` → `(X_train, y_train, X_test, y_test)`
   - Step 5: `train_all_models(X_train, y_train, X_test, y_test)` → results, then rebuild pipelines
   - Steps 6–8: `generate_cv_report`, `evaluate_models`, `generate_comparison_report`
   - Step 9: `explain_top_models` (guarded — skipped if SHAP unavailable)
   - Step 10: `select_and_persist_winner`
   - Step 11: `deploy_winner_model`

7. **Test strategy:** Uses `data_dict` parameter to bypass database. Synthetic OHLCV data with enough rows for feature engineering + label generation + training. Mocks are **not** used for pipeline components — all components execute for real. Only `load_data` is bypassed via the `data_dict` parameter.

      ```python
      def save_dataframes(
          data_dict: dict[str, pd.DataFrame],
          output_dir: str | Path,
      ) -> Path:
          """Save per-ticker DataFrames as Parquet files in *output_dir*.

          Each ticker's DataFrame is written as ``{ticker}.parquet`` using the
          pyarrow engine.  The directory is created if it does not exist; existing
          Parquet files are overwritten.

          Returns the ``Path`` to *output_dir*.
          """
          out = Path(output_dir)
          out.mkdir(parents=True, exist_ok=True)

          for ticker, df in data_dict.items():
              path = out / f"{ticker}.parquet"
              df.to_parquet(path, engine="pyarrow")

          logger.info("Saved %d DataFrames to %s", len(data_dict), out)
          return out


      def load_dataframes(input_dir: str | Path) -> dict[str, pd.DataFrame]:
          """Load per-ticker DataFrames from Parquet files in *input_dir*.

          Returns a dict keyed by ticker symbol (filename stem).

          Raises ``FileNotFoundError`` if *input_dir* does not exist.
          """
          src = Path(input_dir)
          if not src.exists():
              raise FileNotFoundError(f"Directory does not exist: {src}")

          result: dict[str, pd.DataFrame] = {}
          for path in sorted(src.glob("*.parquet")):
              ticker = path.stem
              result[ticker] = pd.read_parquet(path, engine="pyarrow")

          logger.info("Loaded %d DataFrames from %s", len(result), src)
          return result
      ```

      **`_rebuild_pipelines()` helper:**
      Internal function that reconstructs fitted pipelines from TrainingResult list
      by rebuilding and refitting each pipeline on the training data. Needed because
      `train_all_models()` returns results but not fitted pipelines.
      ```python
      def _rebuild_pipelines(
          results: list,
          X_train: np.ndarray,
          y_train: np.ndarray,
      ) -> dict[str, Pipeline]:
      ```
      - Imports model configs (`LINEAR_MODELS`, `TREE_MODELS`, `BOOSTER_MODELS`, `DISTANCE_NEURAL_MODELS`)
      - For each result, builds `Pipeline([scaler, model])`, applies best_params, fits on X_train/y_train
      - Returns dict keyed by `"{model_name}_{scaler_variant}"`

      **`run_training_pipeline()` function:**
      ```python
      def run_training_pipeline(
          tickers: list[str] | None = None,
          db_settings: DBSettings | None = None,
          data_dict: dict[str, pd.DataFrame] | None = None,
          registry_dir: str = "model_registry",
          serving_dir: str = "/models/active",
          target_col: str = "target_7d",
          horizon: int = 7,
          test_ratio: float = 0.2,
          n_splits: int = 5,
          skip_shap: bool = False,
      ) -> PipelineRunResult:
      ```

      Implementation steps (each wrapped in step tracking):

      1. **Initialise run** — generate `run_id`, `started_at`, create `PipelineRunResult`
      2. **Step 1: load_data** — call `load_data(tickers, db_settings)` if `data_dict` is None, otherwise use provided `data_dict`. Record `n_tickers`. Append `"load_data"` to steps_completed.
      3. **Step 2: engineer_features** — call `engineer_features(data_dict)`. Append `"engineer_features"`.
      4. **Step 3: generate_labels** — call `generate_labels(enriched_dict, horizon)`. Append `"generate_labels"`.
      5. **Step 4: prepare_training_data** — call `prepare_training_data(labelled_dict, feature_names, target_col, test_ratio)`. Append `"prepare_training_data"`.
      6. **Step 5: train_all_models** — call `train_all_models(X_train, y_train, X_test, y_test, n_splits)`, then call `_rebuild_pipelines(results, X_train, y_train)`. Record `n_models_trained`. Append `"train_models"`.
      7. **Step 6: generate_cv_report** — call `generate_cv_report(results)`. Append `"cross_validation"`.
      8. **Step 7: evaluate_models** — call `evaluate_models(results)`. Append `"evaluate_models"`.
      9. **Step 8: generate_comparison_report** — call `generate_comparison_report(ranked)`. Append `"model_comparison"`.
      10. **Step 9: explain_top_models** — if not `skip_shap`, call `explain_top_models(results, pipelines, X_test, feature_names, registry_dir)`. Guarded with try/except ImportError. Append `"explainability"`.
      11. **Step 10: select_and_persist_winner** — call `select_and_persist_winner(results, pipelines, feature_names, registry_dir)`. Store return value in `winner_info`. Append `"select_winner"`.
      12. **Step 11: deploy_winner_model** — call `deploy_winner_model(registry_dir, serving_dir)`. Store return value in `deploy_info`. Append `"deploy_model"`.
      13. **Finalise** — set `completed_at`, `status = "completed"`, save run result JSON.

      **Error handling:** Wrap the entire step sequence in try/except. On failure:
      - Set `status = "failed"`, `error = traceback.format_exc()`
      - Save partial run result, then re-raise

      **`_save_run_result()` helper:**

      ```python
      def _save_run_result(result: PipelineRunResult, registry_dir: str) -> Path:
          runs_dir = Path(registry_dir) / "runs"
          runs_dir.mkdir(parents=True, exist_ok=True)
          path = runs_dir / f"pipeline_run_{result.run_id}.json"
          with open(path, "w") as f:
              json.dump(result.to_dict(), f, indent=2, default=str)
          return path
      ```
