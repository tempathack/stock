### Step 2: Create cert-manager.yaml documentation file

File: `k8s/ml/kserve/cert-manager.yaml`

This is a documentation/reference file containing the cert-manager installation
command and verification steps as YAML comments + a stub ConfigMap. KServe
requires cert-manager for its webhook server TLS certificates.

```yaml
# cert-manager installation reference for KServe
# Version: v1.16.2 (compatible with KServe 0.14.x)
#
# Install:
#   kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.16.2/cert-manager.yaml
#
# Verify:
#   kubectl wait --for=condition=Available deployment/cert-manager -n cert-manager --timeout=180s
#   kubectl wait --for=condition=Available deployment/cert-manager-webhook -n cert-manager --timeout=180s
#   kubectl wait --for=condition=Available deployment/cert-manager-cainjector -n cert-manager --timeout=180s
#   kubectl get pods -n cert-manager
#
# Note: cert-manager creates its own 'cert-manager' namespace.
# The upstream manifest is applied directly — no local copy maintained.
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: kserve-cert-manager-ref
  namespace: ml
  labels:
    app: kserve
    app.kubernetes.io/part-of: stock-prediction-platform
data:
  CERT_MANAGER_VERSION: "v1.16.2"
  CERT_MANAGER_URL: "https://github.com/cert-manager/cert-manager/releases/download/v1.16.2/cert-manager.yaml"
```

### Step 3: Create kserve-install.yaml documentation file

File: `k8s/ml/kserve/kserve-install.yaml`

Reference file documenting the KServe installation steps and RawDeployment config patch.

```yaml
# KServe installation reference
# Version: v0.14.1 (latest stable)
# Mode: RawDeployment (no Knative Serving or Istio required)
#
# Prerequisites:
#   - cert-manager v1.16.2+ installed and healthy
#   - MinIO deployed in storage namespace (Phase 51)
#
# Install KServe controller:
#   kubectl apply -f https://github.com/kserve/kserve/releases/download/v0.14.1/kserve.yaml
#   kubectl wait --for=condition=Available deployment/kserve-controller-manager -n kserve --timeout=180s
#
# Install built-in ClusterServingRuntimes (sklearn, xgboost, tensorflow, etc.):
#   kubectl apply -f https://github.com/kserve/kserve/releases/download/v0.14.1/kserve-cluster-resources.yaml
#
# Configure RawDeployment mode (skips Knative Serving + Istio):
#   kubectl patch configmap/inferenceservice-config -n kserve --type=merge \
#     -p '{"data":{"deploy":"{\"defaultDeploymentMode\":\"RawDeployment\"}"}}'
#
# Verify CRDs:
#   kubectl get crd inferenceservices.serving.kserve.io
#   kubectl get crd servingruntimes.serving.kserve.io
#   kubectl get crd clusterservingruntimes.serving.kserve.io
#
# Verify controller:
#   kubectl get pods -n kserve
#   kubectl logs -n kserve deployment/kserve-controller-manager
#
# Note: KServe creates its own 'kserve' namespace.
# The upstream manifests are applied directly — no local copies maintained.
# RawDeployment mode creates standard K8s Deployments + Services instead of
# Knative Services, which is ideal for Minikube/local development.
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: kserve-install-ref
  namespace: ml
  labels:
    app: kserve
    app.kubernetes.io/part-of: stock-prediction-platform
data:
  KSERVE_VERSION: "v0.14.1"
  KSERVE_URL: "https://github.com/kserve/kserve/releases/download/v0.14.1/kserve.yaml"
  KSERVE_RUNTIMES_URL: "https://github.com/kserve/kserve/releases/download/v0.14.1/kserve-cluster-resources.yaml"
  DEPLOY_MODE: "RawDeployment"
```

### Step 4: Create kserve-s3-secret.yaml

File: `k8s/ml/kserve/kserve-s3-secret.yaml`

S3 credentials Secret with KServe annotations so the storage-initializer
init container can authenticate to MinIO.

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: kserve-s3-credentials
  namespace: ml
  labels:
    app: kserve
    app.kubernetes.io/part-of: stock-prediction-platform
  annotations:
    serving.kserve.io/s3-endpoint: minio.storage.svc.cluster.local:9000
    serving.kserve.io/s3-usehttps: "0"
    serving.kserve.io/s3-region: us-east-1
    serving.kserve.io/s3-verifyssl: "0"
type: Opaque
data:
  # Values MUST match MINIO_ROOT_USER / MINIO_ROOT_PASSWORD from k8s/storage/minio-secrets.yaml
  # echo -n 'minioadmin' | base64
  AWS_ACCESS_KEY_ID: <base64-encoded-minio-root-user>
  AWS_SECRET_ACCESS_KEY: <base64-encoded-minio-root-password>
```

Credential values are sourced from MinIO — deploy-all.sh can optionally
auto-generate this from minio-secrets, but to keep it simple the secret
file references the same credential values.
