## Design Decisions
- **DB-backed endpoints** — unlike Phases 20-22 (file-based), market endpoints read from PostgreSQL since market overview needs cross-ticker aggregation
- **Graceful degradation** — `get_market_overview()` returns `[]` and `get_ticker_indicators()` returns `None` when DB or psycopg2 unavailable (no crash)
- **psycopg2 lazy import** — imported inside functions so API starts even without psycopg2 installed (dev convenience)
- **LATERAL join** for overview — efficient single query joining `stocks` with the most recent `ohlcv_daily` row per ticker
- **Indicator reuse** — `get_ticker_indicators()` calls `ml.features.indicators.compute_all_indicators()` directly, avoiding indicator duplication
- **Column normalization** — lowercases all columns before passing to indicator functions (handles yfinance-style uppercase columns)
- **NaN/inf handling** — replaces `np.inf`/`-np.inf` with `NaN`, then `NaN` with `None` for JSON serialization
- **Case-insensitive** ticker lookup in `_load_ohlcv_from_db()` via `.upper()`
