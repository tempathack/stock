Create the weekly training CronJob following the ingestion CronJob pattern:

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: weekly-training
  namespace: ml
  labels:
    app: weekly-training
    app.kubernetes.io/part-of: stock-prediction-platform
    app.kubernetes.io/component: ml-training
spec:
  schedule: "0 3 * * 0"
  concurrencyPolicy: Forbid
  startingDeadlineSeconds: 600
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 2
      activeDeadlineSeconds: 3600
      template:
        metadata:
          labels:
            app: weekly-training
        spec:
          restartPolicy: OnFailure
          containers:
            - name: training
              image: stock-ml-pipeline:latest
              imagePullPolicy: Never
              command: ["python", "-m", "ml.pipelines.training_pipeline"]
              args:
                - "--registry-dir"
                - "$(MODEL_REGISTRY_DIR)"
                - "--serving-dir"
                - "$(SERVING_DIR)"
                - "--skip-shap"
              envFrom:
                - configMapRef:
                    name: ml-pipeline-config
                - secretRef:
                    name: stock-platform-secrets
                    optional: true
              volumeMounts:
                - name: model-artifacts
                  mountPath: /data
              resources:
                requests:
                  cpu: "500m"
                  memory: "1Gi"
                limits:
                  cpu: "2000m"
                  memory: "2Gi"
          volumes:
            - name: model-artifacts
              persistentVolumeClaim:
                claimName: model-artifacts-pvc
```

Key design decisions:
- `schedule: "0 3 * * 0"` = Sunday 03:00 UTC (low-traffic window)
- `concurrencyPolicy: Forbid` prevents overlapping training runs
- `activeDeadlineSeconds: 3600` = 1 hour max training time
- `--skip-shap` in args — SHAP adds significant runtime; skip in scheduled runs
- `$(MODEL_REGISTRY_DIR)` and `$(SERVING_DIR)` are resolved from ConfigMap env vars by K8s
- `secretRef optional: true` — CronJob works even if Secret doesn't exist yet (Phase 36 creates it)
- PVC mounted at /data — all ConfigMap paths (/data/model_registry, /data/models/active, /data/drift_logs) are under this mount

  </action>
</task>

Create the daily drift detection CronJob:

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: daily-drift
  namespace: ml
  labels:
    app: daily-drift
    app.kubernetes.io/part-of: stock-prediction-platform
    app.kubernetes.io/component: ml-drift
spec:
  schedule: "0 22 * * 1-5"
  concurrencyPolicy: Forbid
  startingDeadlineSeconds: 300
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 2
      activeDeadlineSeconds: 1800
      template:
        metadata:
          labels:
            app: daily-drift
        spec:
          restartPolicy: OnFailure
          containers:
            - name: drift-check
              image: stock-ml-pipeline:latest
              imagePullPolicy: Never
              command: ["python", "-m", "ml.drift.trigger"]
              args:
                - "--auto-retrain"
                - "--registry-dir"
                - "$(MODEL_REGISTRY_DIR)"
                - "--serving-dir"
                - "$(SERVING_DIR)"
                - "--drift-log-dir"
                - "$(DRIFT_LOG_DIR)"
                - "--skip-shap"
              envFrom:
                - configMapRef:
                    name: ml-pipeline-config
                - secretRef:
                    name: stock-platform-secrets
                    optional: true
              volumeMounts:
                - name: model-artifacts
                  mountPath: /data
              resources:
                requests:
                  cpu: "200m"
                  memory: "512Mi"
                limits:
                  cpu: "1000m"
                  memory: "1Gi"
          volumes:
            - name: model-artifacts
              persistentVolumeClaim:
                claimName: model-artifacts-pvc
```

Key design decisions:
- `schedule: "0 22 * * 1-5"` = weekdays 22:00 UTC (after market close + ingestion)
- `activeDeadlineSeconds: 1800` = 30 min max (drift check is fast; retrain if triggered takes longer)
- `--auto-retrain` flag tells trigger.py to run drift check and auto-trigger retraining if drift detected
- Lower resource limits than training since drift check alone is lightweight; if retrain triggers, the same container handles it
- Uses the same PVC and ConfigMap as the training CronJob

  </action>
</task>
