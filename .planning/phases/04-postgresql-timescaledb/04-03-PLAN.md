Run all 7 DB requirement smoke tests against the live PostgreSQL pod. Execute each command and record the output:

DB-02 — TimescaleDB extension enabled:
```bash
kubectl exec -n storage deploy/postgresql -- psql -U stockuser -d stockdb -c "SELECT extname FROM pg_extension WHERE extname='timescaledb';"
```
Expected: row showing `timescaledb`

DB-03 — All 6 tables exist:
```bash
kubectl exec -n storage deploy/postgresql -- psql -U stockuser -d stockdb -c "SELECT tablename FROM pg_tables WHERE schemaname='public' ORDER BY tablename;"
```
Expected: 6 rows: drift_logs, model_registry, ohlcv_daily, ohlcv_intraday, predictions, stocks

DB-04 — Composite PKs on ohlcv tables:
```bash
kubectl exec -n storage deploy/postgresql -- psql -U stockuser -d stockdb -c "SELECT conname, contype FROM pg_constraint WHERE conrelid IN ('ohlcv_daily'::regclass, 'ohlcv_intraday'::regclass) AND contype='p';"
```
Expected: 2 rows with PK constraint names

DB-05 — Indexes on ticker columns:
```bash
kubectl exec -n storage deploy/postgresql -- psql -U stockuser -d stockdb -c "SELECT indexname, tablename FROM pg_indexes WHERE tablename IN ('ohlcv_daily','ohlcv_intraday','predictions','drift_logs','model_registry') ORDER BY tablename, indexname;"
```
Expected: multiple rows including idx_* indexes and PK indexes

DB-06 — Hypertables with correct chunk intervals:
```bash
kubectl exec -n storage deploy/postgresql -- psql -U stockuser -d stockdb -c "SELECT hypertable_name, chunk_time_interval FROM timescaledb_information.hypertables ORDER BY hypertable_name;"
```
Expected: 2 rows — ohlcv_daily (chunk_time_interval=1 mon), ohlcv_intraday (chunk_time_interval=1 day)

DB-01+DB-07 — Pod + PVC + K8s resources summary:
```bash
kubectl get pods,pvc,secret,configmap -n storage
```
Expected: postgresql pod Running, postgresql-pvc Bound, stock-platform-secrets and postgresql-init-sql present

If any command fails:
- "function create_hypertable does not exist" → CREATE EXTENSION was not picked up; check if PVC had stale data (delete PVC and re-init per Task 1 fallback)
- "relation does not exist" → init.sql had a FK ordering error; check init.sql table creation order
- "relation ohlcv_daily already exists and is not a table" → hypertable creation conflict; check if_not_exists parameter
