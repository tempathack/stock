**Section 2 — ohlcv_daily (90 days × 20 tickers = ~1800 rows):**

Generate using a `generate_series` approach in SQL. For each ticker, use a deterministic starting price based on ticker index (AAPL~175, MSFT~420, etc.), then use a random walk with small daily changes:

```sql
-- Use generate_series for dates, seed per-ticker for determinism
INSERT INTO ohlcv_daily (ticker, date, open, high, low, close, adj_close, volume, vwap)
SELECT
    t.ticker,
    d.date,
    -- open: previous close ± small random offset
    round((base_price + (random_offset * 0.02 * base_price))::numeric, 4) as open,
    -- high: max(open, close) + small positive offset
    ...
```

Actually, for simplicity and determinism, generate the daily data using a **heredoc with `awk`** or a **PL/pgSQL DO block** that creates synthetic but realistic OHLCV:

```sql
DO $$
DECLARE
    tickers text[] := ARRAY['AAPL','MSFT','GOOGL','AMZN','NVDA','META','TSLA','BRK-B','JPM','JNJ','V','PG','UNH','HD','MA','BAC','XOM','PFE','ABBV','CVX'];
    base_prices numeric[] := ARRAY[178.50, 425.30, 175.20, 185.40, 880.50, 510.20, 245.80, 415.60, 198.70, 155.30, 280.40, 162.50, 530.20, 355.80, 465.30, 38.50, 108.20, 27.80, 175.40, 152.30];
    t text;
    bp numeric;
    cur_price numeric;
    day_open numeric;
    day_high numeric;
    day_low numeric;
    day_close numeric;
    day_vol bigint;
    d date;
BEGIN
    FOR i IN 1..array_length(tickers, 1) LOOP
        t := tickers[i];
        bp := base_prices[i];
        cur_price := bp;
        FOR d IN SELECT generate_series(CURRENT_DATE - INTERVAL '90 days', CURRENT_DATE - INTERVAL '1 day', '1 day')::date LOOP
            -- Skip weekends
            IF EXTRACT(DOW FROM d) IN (0, 6) THEN CONTINUE; END IF;
            -- Daily change: -2% to +2% deterministic via date/ticker hash
            day_open := round(cur_price * (1 + (hashtext(t || d::text) % 200 - 100)::numeric / 10000), 4);
            day_close := round(cur_price * (1 + (hashtext(d::text || t) % 200 - 100)::numeric / 10000), 4);
            day_high := round(greatest(day_open, day_close) * (1 + abs(hashtext(t || 'h' || d::text) % 50)::numeric / 10000), 4);
            day_low := round(least(day_open, day_close) * (1 - abs(hashtext(t || 'l' || d::text) % 50)::numeric / 10000), 4);
            day_vol := 1000000 + abs(hashtext(t || 'v' || d::text) % 50000000);
            INSERT INTO ohlcv_daily (ticker, date, open, high, low, close, adj_close, volume, vwap)
            VALUES (t, d, day_open, day_high, day_low, day_close, day_close, day_vol, round((day_high + day_low + day_close) / 3, 4))
            ON CONFLICT (ticker, date) DO UPDATE SET
                open = EXCLUDED.open, high = EXCLUDED.high, low = EXCLUDED.low,
                close = EXCLUDED.close, adj_close = EXCLUDED.adj_close,
                volume = EXCLUDED.volume, vwap = EXCLUDED.vwap;
            cur_price := day_close;
        END LOOP;
    END LOOP;
END $$;
```

**Section 5 — predictions (140 rows: 20 tickers × 7 days):**

```sql
-- Get the active model ID
DO $$
DECLARE
    active_model_id integer;
    tickers text[] := ARRAY['AAPL','MSFT',...,'CVX'];
    t text;
    last_close numeric;
    pred_price numeric;
    d integer;
BEGIN
    SELECT model_id INTO active_model_id FROM model_registry WHERE is_active = true LIMIT 1;
    IF active_model_id IS NULL THEN RETURN; END IF;
    
    FOR i IN 1..array_length(tickers, 1) LOOP
        t := tickers[i];
        SELECT close INTO last_close FROM ohlcv_daily WHERE ticker = t ORDER BY date DESC LIMIT 1;
        IF last_close IS NULL THEN CONTINUE; END IF;
        
        FOR d IN 1..7 LOOP
            pred_price := round(last_close * (1 + (hashtext(t || d::text || 'pred') % 100 - 30)::numeric / 10000), 4);
            INSERT INTO predictions (ticker, prediction_date, predicted_date, predicted_price, model_id, confidence)
            VALUES (t, CURRENT_DATE, CURRENT_DATE + d, pred_price, active_model_id, round(0.55 + (abs(hashtext(t || d::text)) % 30)::numeric / 100, 4));
        END LOOP;
    END LOOP;
END $$;
```

## Design Decisions
- **Pure SQL via psql** — No Python dependency; script works from any machine with psql client installed
- **PL/pgSQL DO blocks** for OHLCV — Generates thousands of rows in-database without round-tripping
- **hashtext() for determinism** — PostgreSQL hash function ensures same data on every run (no `random()`)
- **ON CONFLICT for idempotency** — Safe to re-run; updates existing rows rather than failing
- **Environment variable connection** — Matches K8s ConfigMap pattern; works with port-forward or direct connection
- **Real company data** — Actual S&P 500 tickers, company names, sectors for realistic API responses
- **Price realism** — Base prices approximate real 2026 ranges; OHLC relationship invariants maintained (low ≤ open/close ≤ high)
