Create `tests/integration/test_drift_cycle.py` with:

**Fixtures:**
- `synthetic_data_dict` — Same as Plan 03 (3 tickers × 200 rows synthetic OHLCV); can import from conftest if shared
- `baseline_environment(tmp_path, synthetic_data_dict)` — A session-scoped-ish fixture that:
  1. Runs `run_training_pipeline(data_dict=..., registry_dir=tmp_path/"registry", serving_dir=tmp_path/"serving", skip_shap=True)`
  2. Returns a dict with `registry_dir`, `serving_dir`, `data_dict`, and `result` (PipelineRunResult)
  3. This establishes the "before drift" baseline
- `drifted_features(synthetic_data_dict)` — Creates shifted feature distributions:
  - Takes the synthetic data, engineers features via `engineer_features()`, splits into reference (first 70%) and current (last 30%)
  - Shifts `current` features by adding a systematic offset (e.g., multiply numeric columns by 1.3) to simulate distribution shift
  - Returns `(reference_features, current_features)`
- `drifted_errors()` — Creates error arrays that will trigger prediction drift:
  - `baseline_errors`: `np.random.RandomState(42).normal(0, 0.02, 100)` (small errors)
  - `recent_errors`: `np.random.RandomState(42).normal(0.05, 0.04, 100)` (larger, biased errors)
  - Returns `(baseline_errors, recent_errors, historical_rmse, recent_rmse)` where recent_rmse > historical_rmse × 1.5

**Test class `TestDriftCycle`:**

**Test 1: `test_drift_detection_triggers`**
- Call `DriftMonitor().check(reference_features, current_features, baseline_errors, recent_errors, historical_rmse, recent_rmse)`
- Assert `check_result.any_drift is True`
- Assert at least one of `data_drift`, `prediction_drift`, or `concept_drift` has `is_drifted=True`

**Test 4: `test_retraining_produces_new_model`**

**Test 5: `test_serving_directory_updated_after_retrain`**

## Design Decisions
- **Real pipeline execution** — No mocking of training; the actual ML models are trained to validate the full cycle
- **Synthetic drift injection** — Feature shift + error inflation guarantees drift detection triggers
- **Shared baseline fixture** — Train once, run multiple drift tests against the same baseline (saves ~5min)
- **@pytest.mark.slow** — These tests train real models twice; CI can skip with `-m "not slow"`
- **Idempotency check** — Ensures the drift cycle can run multiple times without corrupting state
- **No database dependency** — Uses data_dict parameter and file-based artifacts throughout
