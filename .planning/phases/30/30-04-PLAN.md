## Design Decisions
- **TestClient, not httpx** — FastAPI's TestClient runs in-process (no server needed), uses real middleware and routing
- **File fixture seeding** — Mimics real deployment: prediction_service reads from files, not DB; fixtures match that pattern
- **Mock only external services** — Yahoo Finance and Kafka are mocked; market_service is mocked at function level since it needs PostgreSQL
- **Contract validation over logic testing** — These tests verify response shapes, not business logic (that's covered by unit tests)
- **snake_case assertion** — Ensures API responses are directly consumable by the frontend TypeScript layer
