## Spacing Scale

Declared values (must be multiples of 4). Source: MUI default spacing unit = 8px; existing pages use `sx={{ p: 2 }}` (16px), `sx={{ gap: 2 }}` (16px), `sx={{ mb: 3 }}` (24px).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding, inline badges |
| sm | 8px | Compact element spacing, list item padding |
| md | 16px | Default card padding (`p: 2`), grid column gaps |
| lg | 24px | Section vertical spacing (`mb: 3`), panel headers |
| xl | 32px | Layout gaps between major panel rows (`gap: 4`) |
| 2xl | 48px | Major section breaks, page-level top padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Sidebar drawer width: 220px (locked by `DRAWER_WIDTH` in `Sidebar.tsx` — do not change)
- Status chip height: 24px (MUI Chip `size="small"`) — not a spacing exception, standard MUI
- Touch targets: minimum 44px tall for interactive controls at 375px breakpoint

Source: `Sidebar.tsx`, `Drift.tsx`, existing `<Container>` + `<Grid>` patterns.

| Role | Value | Usage |

## Interaction Contract

### Polling behavior

| Panel | Interval | React Query key | Stale while revalidate |
|-------|----------|-----------------|----------------------|
| SystemHealthSummary | 30s | `["analytics", "summary"]` | yes |
| StreamHealthPanel | 10s | `["analytics", "flink", "jobs"]` | yes |
| FeatureFreshnessPanel | 30s | `["analytics", "feast", "freshness"]` | yes |
| OLAPCandleChart | on-demand (interval change) | `["market", "candles", ticker, interval]` | yes |
| StreamLagMonitor | 15s | `["analytics", "kafka", "lag"]` | yes |

All hooks use `retry: false` — a failed poll shows the last cached value with a stale indicator, not an error explosion.

### Status chip colors (StatusChip pattern)

| State | MUI color prop | Hex |
|-------|---------------|-----|
| RUNNING | `color="success"` | `#4caf50` |
| FAILED / FAILING | `color="error"` | `#f44336` |
| RESTARTING / CANCELLING | `color="warning"` | `#ff9800` |
| UNKNOWN | `color="default"` | `#9fa8da` |

### Staleness thresholds (FeatureFreshnessPanel)

| Staleness | Color | LinearProgress color |
|-----------|-------|----------------------|
| < 1 hour | green | `color="success"` |
| 1–6 hours | amber | `color="warning"` |
| > 6 hours | red | `color="error"` |

### OLAPCandleChart interval toggle

- Default interval: `1h`
- Supported intervals: `["1h", "1d"]` — only these two (Phase 64 created continuous aggregates for 1h and 1d only; 5m and 4h do not exist)
- `<ToggleButtonGroup exclusive value={interval} onChange={...}>` with MUI
- Active toggle: `color="primary"` border + text

### Empty states (when backend returns empty array or service unreachable)

| Panel | Empty state — icon | Empty state heading | Empty state body |
|-------|-------------------|---------------------|-----------------|
| StreamHealthPanel | `StreamIcon` | "No stream jobs detected" | "Flink may be unreachable inside the cluster. Use kubectl port-forward to expose the Flink REST API." |
| FeatureFreshnessPanel | `DataObjectIcon` | "No feature views found" | "Feast registry may not be initialized. Run feast apply to register feature views." |
| OLAPCandleChart | `CandlestickChartIcon` | "No candle data" | "No OHLCV data for this interval. Try a different interval or ingest historical data first." |
| StreamLagMonitor | `SpeedIcon` | "Lag data unavailable" | "Consumer group metrics could not be retrieved. Check Kafka connectivity." |
| SystemHealthSummary | — | Show each metric card with value "—" | No heading needed — individual card shows "N/A" |

Use `<PlaceholderCard>` with the above copy when `data` is an empty array and `isLoading` is false.

### Error states (API returned non-200 or threw)

All panels: use `<Alert severity="error" sx={{ mt: 1 }}>` inside the panel body — do not use full-page error.

Error copy pattern: `"{service} is currently unreachable — displaying last known data."`

Examples:
- `"Flink REST API is currently unreachable — displaying last known data."`
- `"Feast registry query failed — check FastAPI logs."`
- `"Kafka lag endpoint returned an error — retrying in 15s."`

### Error boundary

Wrap each of the 5 panels individually with `<ErrorBoundary FallbackComponent={ErrorFallback}>` — one panel crashing must not take down the page. Use the existing `ErrorFallback` from `components/ui/ErrorFallback.tsx`.

## Accessibility

- All status chips include `aria-label="{state} status"` (e.g. `aria-label="RUNNING status"`)
