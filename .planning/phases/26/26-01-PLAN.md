## Design Decisions

- **Recharts over D3 direct** — React-native component API, less boilerplate for standard chart types, good enough for dashboard charts; D3 would be overkill
- **No pagination for model table** — typical training runs produce 15-20 models; fits in a single scrollable view without pagination complexity
- **Client-side sorting over API** — small dataset (< 50 rows), sorting is instant client-side; no need for API query params
- **onSelectModel callback** — enables Plan 26-02 to sync chart focus with table row clicks without prop drilling (state in parent Models.tsx)
- **Fold stability as a column** — makes it immediately visible which models are unstable (high variance) without expanding a row
