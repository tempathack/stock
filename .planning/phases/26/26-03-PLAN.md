## Design Decisions

- **ModelDetailPanel as separate component** — keeps table simple (row rendering only) while detail is extracted into a focused component
- **Hyperparameters as DL** — definition lists are semantically correct for key-value metadata; `font-mono` for values aids readability
- **Color-coded fold stability** — instantly identifies unstable models (high fold variance means unreliable OOS generalization)
- **No accordion/expand in table rows** — detail panel below table is cleaner for complex content (charts, multi-section metadata); row expanding in tables causes layout jank
- **Empty state with pipeline guidance** — helps users understand that models appear after the training pipeline runs; reduces confusion for fresh deployments
