## Plan 56-01 — KServe V2 Inference Client & Prediction Service Refactor

### Objective
Create a thin async HTTP client (`kserve_client.py`) that speaks the KServe V2/Open Inference Protocol, then refactor `prediction_service.py` to delegate inference to KServe when `KSERVE_ENABLED=True`, falling back to cached predictions on failure.

### Requirements Addressed
- **KSERV-09**: prediction_service.py calls KServe V2 inference endpoint via HTTP
- **KSERV-10**: KSERVE_INFERENCE_URL configurable via K8s ConfigMap
- **KSERV-11**: Prediction API response schema backward-compatible with pre-KServe format

### Why
The current prediction path loads `pipeline.pkl` from disk (PVC mount) and runs `pipeline.predict()` in-process. With KServe InferenceService (deployed in Phase 55), the model is served externally via MLServer with autoscaling, health checking, and the standard V2 protocol. The API pod no longer needs sklearn/pickle dependencies or PVC mounts for model files — it just sends HTTP requests to KServe.

### Scope
| In Scope | Out of Scope |
|----------|-------------|
| `kserve_client.py` async HTTP client | A/B testing canary changes (plan 56-02) |
| `config.py` KServe settings | Frontend changes (none needed) |
| `prediction_service.py` dual-path logic | KServe InferenceService creation (phase 55) |
| `configmap.yaml` env vars | gRPC protocol support |
| `main.py` shutdown hook | Batch inference optimization |
| `requirements.txt` httpx dep | Feature store integration |

### Key Design Decisions
1. **Feature flag (`KSERVE_ENABLED`)** — allows instant rollback via ConfigMap without code changes
2. **httpx over aiohttp** — lighter weight, matches FastAPI ecosystem, already used elsewhere
3. **features.json still read from disk** — needed for column alignment; file is tiny and avoids an extra KServe metadata call
4. **Fallback returns None** — existing router logic already handles None by falling back to cached predictions
5. **No batch endpoint** — KServe V2 supports batching but individual ticker calls suffice at our scale; batch optimization deferred
