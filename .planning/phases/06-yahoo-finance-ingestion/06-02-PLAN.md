Implement `kafka_producer.py` with `OHLCVProducer` class:

```python
"""Kafka producer service for publishing OHLCV data to topics."""
from __future__ import annotations

import json

from confluent_kafka import Producer

from app.config import settings
from app.utils.logging import get_logger

logger = get_logger(__name__)
```

**Class `OHLCVProducer`:**

- `__init__(self, producer: Producer | None = None)`: If `producer` is provided, use it (for testing). Otherwise create a real `Producer` with:
  ```python
  {
      "bootstrap.servers": settings.KAFKA_BOOTSTRAP_SERVERS,
      "acks": "all",
      "retries": 3,
      "retry.backoff.ms": 100,
      "linger.ms": 5,
      "batch.num.messages": 100,
  }
  ```
  Store topic names: `self._intraday_topic = settings.INTRADAY_TOPIC` and `self._historical_topic = settings.HISTORICAL_TOPIC`.

- `_delivery_report(self, err, msg)`: Static or instance method. If `err is not None`: `logger.error("kafka_delivery_failed", error=str(err), topic=msg.topic())`. If `err is None`: `logger.debug("kafka_delivery_ok", topic=msg.topic(), partition=msg.partition(), offset=msg.offset())`. Never raises.

- `_get_topic(self, fetch_mode: str) -> str`: Return `self._intraday_topic` if `fetch_mode == "intraday"` else `self._historical_topic`.

- `produce_records(self, records: list[dict]) -> int`: Produce all records to Kafka. For each record:
  1. Determine topic via `_get_topic(record["fetch_mode"])`
  2. Serialize: `key = record["ticker"].encode("utf-8")`, `value = json.dumps(record).encode("utf-8")`
  3. `self._producer.produce(topic=topic, key=key, value=value, callback=self._delivery_report)`

  Group records by ticker. After producing all records for a ticker, call `self._producer.flush(timeout=30)`. Log INFO: `event="batch_produced"`, `ticker=ticker`, `count=N`, `topic=topic`.

  If `records` is empty, return 0 immediately.

  Return total number of records produced.

- `flush(self, timeout: float = 30) -> None`: Call `self._producer.flush(timeout=timeout)`. Utility method for external callers.

Key implementation details:
- Accept `producer` param in `__init__` for dependency injection (testability)
- The `_delivery_report` method must NOT re-raise errors -- it only logs
- `produce_records` groups by ticker using a simple dict or itertools.groupby to batch flushes properly
- If all records have the same ticker (common case), only one flush call
- Use `from __future__ import annotations` at top
