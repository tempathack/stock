# Plan 18-01 — Data Preparation & Training Orchestration Component

## Objective
Add `prepare_training_data()` and `train_all_models_pipeline()` to `ml/pipelines/components/model_trainer.py` — step 4 (KF-05) of the Kubeflow training pipeline. `prepare_training_data()` consolidates multi-ticker DataFrames into a single temporal train/test numpy split. `train_all_models_pipeline()` is the pipeline entry point that trains all model families and returns fitted pipelines for downstream use. ## Design Decisions

1. **`prepare_training_data()` — consolidation + temporal split:**
   - Concatenates all per-ticker DataFrames into one large DataFrame, sorted by DatetimeIndex
   - Extracts target column (default `target_7d`) as `y`, remainder as `X`
   - Splits at `int(len(df) * (1 - test_ratio))` — last `test_ratio` fraction is the test set
   - Returns `(X_train, y_train, X_test, y_test)` as numpy `ndarray`s
   - Raises `ValueError` if input dict is empty or all DataFrames are empty
   - `target_col` parameter defaults to `"target_7d"` for flexibility

2. **`train_all_models_pipeline()` — pipeline entry point:**
   - Takes `data_dict: dict[str, pd.DataFrame]` and `feature_names: list[str]`
   - Also accepts `target_col`, `test_ratio`, `n_splits` parameters
   - Calls `prepare_training_data()` to get numpy arrays
   - Calls existing `train_all_models(X_train, y_train, X_test, y_test, n_splits)` for results
   - Reconstructs fitted pipelines for each result using `_build_pipeline()` + `fit(X_train, y_train)`
     with best_params applied via `pipeline.set_params(**{f"model__{k}": v for k, v in best_params.items()})`
   - Returns `(list[TrainingResult], dict[str, Pipeline])` where dict keys are `"{model_name}_{scaler_variant}"`
   - This signature matches what `model_selector.select_and_persist_winner()` and `explainer.explain_top_models()` expect

3. **Pipeline reconstruction uses `TrainingResult.best_params`:** No refactor of `train_single_model()` needed. After `train_all_models()` returns, each result's `best_params` are applied to a fresh pipeline, which is then fit on the full `X_train`. This produces a pipeline identical to the one found by `RandomizedSearchCV`.

4. **Fixture strategy for tests:** Tests need a realistic `data_dict` from the pipeline. A `pipeline_data` fixture runs `engineer_features()` + `generate_labels()` on the conftest `sample_ohlcv_df` fixture. Since this fixture is computationally heavy (~250 rows through all indicators + lags), it's session-scoped (or narrow-scoped with caching via `@pytest.fixture`). The resulting data_dict will have ~40-50 rows per ticker, which is enough for 3-fold CV with small model configs.

5. **Fast training in tests:** Tests use `n_splits=3` and only train a small subset (linear models only via `train_linear_models`) to keep runtime under 30 seconds. A dedicated integration test for `train_all_models_pipeline()` uses `n_splits=3`.

      ```python
      def train_all_models_pipeline(
          data_dict: dict[str, pd.DataFrame],
          feature_names: list[str],
          target_col: str = "target_7d",
          test_ratio: float = 0.2,
          n_splits: int = 5,
      ) -> tuple[list[TrainingResult], dict[str, Pipeline]]:
          """Pipeline entry point: prepare data, train all models, return results + pipelines.

          Parameters
          ----------
          data_dict:
              Per-ticker DataFrames from ``label_generator``.
          feature_names:
              Ordered list of feature column names.
          target_col:
              Name of the target column.
          test_ratio:
              Fraction of data reserved for the test set.
          n_splits:
              Number of TimeSeriesSplit folds.

          Returns
          -------
          ``(results, pipelines)`` — sorted results and fitted pipelines keyed by
          ``"{model_name}_{scaler_variant}"``.
          """
      ```
      Implementation:
      - Call `prepare_training_data(data_dict, feature_names, target_col, test_ratio)` → X_train, y_train, X_test, y_test
      - Call `train_all_models(X_train, y_train, X_test, y_test, n_splits)` → results
      - Build `pipelines: dict[str, Pipeline]` by iterating results:
        - Look up `ModelConfig` from the combined config dicts (`LINEAR_MODELS | TREE_MODELS | BOOSTER_MODELS | DISTANCE_NEURAL_MODELS`)
        - Call `_build_pipeline(config, result.scaler_variant)` to get a fresh pipeline
        - If `result.best_params`, apply via `pipeline.set_params(**{f"model__{k}": v for k, v in result.best_params.items()})`
        - Fit on full `X_train, y_train`
        - Store as `pipelines[f"{result.model_name}_{result.scaler_variant}"]`
      - Return `(results, pipelines)`
      - Log: total models trained, pipeline count
    </file>
