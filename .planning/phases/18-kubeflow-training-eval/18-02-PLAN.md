## Objective
Add `generate_cv_report()` to `ml/pipelines/components/evaluator.py` — the KF-06 component of the Kubeflow training pipeline. This function extracts per-model, per-fold cross-validation metrics from `TrainingResult` objects and produces a structured report suitable for downstream consumption (API, frontend, pipeline artifacts). Full test coverage. Satisfies **KF-06**.

## Dependencies
- `ml/models/model_configs.py` — `TrainingResult` dataclass with `fold_metrics: list[dict[str, float]]` and `fold_stability: float` (Phases 12–14)
- `ml/pipelines/components/evaluator.py` — existing `evaluate_models()` and `generate_comparison_report()` (Phase 15)
- `ml/tests/test_evaluator.py` — existing test pattern using synthetic `_make_result()` helper

## Design Decisions

1. **`generate_cv_report()` extracts fold-level detail:** Each `TrainingResult` already contains `fold_metrics` (a list of per-fold metric dicts) and `fold_stability` (std of RMSE across folds). The CV report restructures this into a format showing per-model fold performance, mean/std per metric, and aggregate statistics across all models.

2. **Report structure:**
   ```python
   {
       "total_models": int,
       "n_folds": int,
       "models": [
           {
               "model_name": str,
               "scaler_variant": str,
               "fold_metrics": [...],       # raw per-fold dicts
               "mean_metrics": {...},        # mean of each metric across folds
               "std_metrics": {...},         # std of each metric across folds
               "fold_stability": float,      # from TrainingResult
           },
           ...
       ],
       "aggregate": {
           "best_cv_rmse": {"model": str, "value": float},
           "most_stable": {"model": str, "value": float},
           "least_stable": {"model": str, "value": float},
       }
   }
   ```

3. **`n_folds` derived from data:** Taken from `len(results[0].fold_metrics)`. Raises `ValueError` if results is empty.

4. **Mean/std computation:** Uses `numpy.mean` and `numpy.std` across fold dicts for each metric key. Only includes keys present in all folds.

5. **Aggregate statistics:** Identifies the model with the best (lowest) mean CV RMSE, the most stable (lowest `fold_stability`), and the least stable (highest `fold_stability`). Uses `"{model_name}_{scaler_variant}"` as the model key.

6. **No changes to existing functions:** `evaluate_models()` and `generate_comparison_report()` remain untouched. `generate_cv_report()` is additive.
