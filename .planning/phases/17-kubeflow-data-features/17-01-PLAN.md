# Plan 17-01 ŌĆö Data Loading Component + KFP Scaffold

## Objective
Implement `ml/pipelines/components/data_loader.py` ŌĆö the first step of the Kubeflow training pipeline. This component connects to PostgreSQL, loads OHLCV daily data for a given list of tickers, and returns per-ticker DataFrames ready for feature engineering. Scaffold the KFP standalone deployment manifest. Full test coverage. Satisfies **KF-01** (scaffold) and **KF-02**.

## Dependencies
- `db/init.sql` ŌĆö `ohlcv_daily` table schema (ticker, date, open, high, low, close, adj_close, volume, vwap)
- `services/kafka-consumer/consumer/config.py` ŌĆö DB connection pattern reference
- `k8s/storage/configmap.yaml` ŌĆö `POSTGRES_DB`, `POSTGRES_USER` config values
- `ml/tests/conftest.py` ŌĆö `sample_ohlcv_df` fixture

## Design Decisions

1. **DBSettings dataclass:** Environment-variable-driven configuration with sensible K8s defaults. Uses `os.environ.get()` ŌĆö no pydantic dependency in the `ml/` package.

2. **Connection management:** Uses `psycopg2.connect()` directly (not a pool) ŌĆö pipeline components are short-lived batch jobs, not long-running services. Connection opened per `load_data()` call, closed on exit.

3. **Query strategy:** Single query per ticker using parameterized SQL (no injection risk). Optional `start_date`/`end_date` filtering. Results ordered by `date ASC` for time-series consistency.

4. **DataFrame contract:** Each returned DataFrame has a `pd.DatetimeIndex` (from the `date` column) and columns: `open, high, low, close, volume`. The `adj_close` and `vwap` columns are excluded from the ML pipeline features (not used by any indicator).

5. **Empty-ticker handling:** If a ticker returns zero rows from the DB, it is logged as a warning and omitted from the result dict (not an error).

6. **KFP manifest:** Scaffolds `k8s/ml/kubeflow/kfp-standalone.yaml` with a comment-annotated reference to the official KFP v2 standalone deployment, pinned to version 2.3.0. Not applied to cluster in this phase.

      ```python
      import os
      from dataclasses import dataclass, field
      from datetime import date
      import pandas as pd
      import psycopg2
      ```

      **Class: `DBSettings`**
      ```python
      @dataclass
      class DBSettings:
          """PostgreSQL connection settings with env-var defaults."""
          host: str = field(default_factory=lambda: os.environ.get("POSTGRES_HOST", "postgresql.storage.svc.cluster.local"))
          port: int = field(default_factory=lambda: int(os.environ.get("POSTGRES_PORT", "5432")))
          database: str = field(default_factory=lambda: os.environ.get("POSTGRES_DB", "stockdb"))
          user: str = field(default_factory=lambda: os.environ.get("POSTGRES_USER", "stockuser"))
          password: str = field(default_factory=lambda: os.environ.get("POSTGRES_PASSWORD", "devpassword123"))

          @classmethod
          def from_env(cls) -> "DBSettings":
              return cls()

          @property
          def connection_string(self) -> str:
              return f"postgresql://{self.user}:{self.password}@{self.host}:{self.port}/{self.database}"
      ```

      **Functions:**

      ```python
      def load_ticker_data(
          ticker: str,
          conn,
          start_date: date | None = None,
          end_date: date | None = None,
      ) -> pd.DataFrame:
      ```
      - Builds parameterized SELECT query for `ohlcv_daily` WHERE ticker = %s
      - Appends date range filters if provided (AND date >= %s, AND date <= %s)
      - ORDER BY date ASC
      - Uses `cursor.execute(sql, params)` ŌĆö never string interpolation
      - Converts result to DataFrame with columns [open, high, low, close, volume]
      - Sets `date` column as DatetimeIndex
      - Returns empty DataFrame with correct columns if no rows

      ```python
      def load_data(
          tickers: list[str],
          settings: DBSettings | None = None,
          start_date: date | None = None,
          end_date: date | None = None,
      ) -> dict[str, pd.DataFrame]:
      ```
      - Validates tickers is a list (raises TypeError otherwise)
      - Creates DBSettings from env if not provided
      - Opens psycopg2 connection using settings
      - Iterates tickers, calls `load_ticker_data()` for each
      - Skips empty results with warning log
      - Closes connection in `finally` block
      - Returns `dict[str, pd.DataFrame]`
