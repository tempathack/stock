## Steps

### 1. Implement `compute_lag_features(df, lags=None)`
- Default lags: `[1, 2, 3, 5, 7, 14, 21]`
- For each lag `n`: `df[f"lag_close_{n}"] = df["close"].shift(n)`
- Returns DataFrame with new columns appended

### 2. Implement `compute_rolling_stats(df, windows=None)`
- Default windows: `[5, 10, 21]`
- For each window `w`, compute 4 stats on `close`:
  - `df[f"rolling_mean_{w}"] = df["close"].rolling(w).mean()`
  - `df[f"rolling_std_{w}"] = df["close"].rolling(w).std()`
  - `df[f"rolling_min_{w}"] = df["close"].rolling(w).min()`
  - `df[f"rolling_max_{w}"] = df["close"].rolling(w).max()`
- No `center=True` — trailing windows only

### 3. Implement `generate_target(df, horizon=7)`
- `df[f"target_{horizon}d"] = df["close"].shift(-horizon) / df["close"] - 1`
- Percentage return target: `(close_t+7 - close_t) / close_t`
- Last `horizon` rows will be NaN (no future data)

### 4. Implement `drop_incomplete_rows(df)`
- `return df.dropna().copy()`
- Drops all rows with any NaN (warm-up + tail)

### 5. Write comprehensive tests in `ml/tests/test_lag_features.py`
- TestLagFeatures: column presence, shift correctness, custom lags, no NaN in valid range, preserves OHLCV
- TestRollingStats: column presence, rolling mean matches manual, custom windows, warm-up NaNs, no center bias
- TestTarget: column presence, percentage return correctness, tail NaNs, no-leakage verification
- TestDropIncomplete: all NaNs removed, row count reduced, no data loss for complete rows

### 6. Update `ml/features/__init__.py`
- Export new functions
