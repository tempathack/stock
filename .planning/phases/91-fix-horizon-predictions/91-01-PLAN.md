# Phase 91 — Plan 01: Backend horizon scaling (prediction_service.py)

## Goal
All four forecast horizons (1d / 7d / 14d / 30d) return meaningfully different
predicted prices and return percentages.

## Root Cause
The DB fallback in `load_db_predictions` silently dropped the horizon filter
(lines 234-236) and recursively returned *all* predictions regardless of horizon,
so every tab received identical data. Live inference and cached-file paths had
the same gap when horizon-specific models/files were absent.
