<objective>
Register fred_macro_fv in the Feast feature repository and extend _TRAINING_FEATURES to include all 14 FRED series. For inference, extend _fetch_from_feast() in the API feast_online_service to make a second get_online_features(entity_rows=[{"ticker": "MACRO"}]) call for FRED features, merging them into the existing response. This uses the same Feast online store path as stock features (ticker='MACRO' column already added to feast_fred_macro in Wave 2). This turns all Plan 01 RED tests GREEN.

Purpose: Without this plan, feast_fred_macro exists but is invisible to training — Feast cannot join FRED features to the entity_df because the FeatureView is not registered. The MACRO entity_rows approach lets FRED features flow through the same Feast online store path as ticker-keyed stock features.
Output: fred_macro_fv in feature_repo.py (49 total features in _TRAINING_FEATURES), FRED features merged into _fetch_from_feast() in feast_online_service.py.
</objective>
