# Plan 96-05: SKTIME_REGRESSION_MODELS Family Definition, Trainer Integration & Validation

## Goal
Register the 5 sktime regression wrappers as a `SKTIME_REGRESSION_MODELS` family in
`model_configs.py`, wire them into `model_trainer.py` with a `train_sktime_regression_models()`
function, expose a `--skip-sktime-regression` CLI flag, and run a full end-to-end training
validation confirming all models appear in the model registry with valid OOS metrics.

---

## Context

From Plan 96-04 we have 5 sklearn-compatible regression wrappers:
- `MiniRocketWrapper` — requires numba (fast JIT)
- `RocketWrapper` — requires numba
- `TimeSeriesForestWrapper` — no extra deps
- `RandomIntervalWrapper` — no extra deps
- `Catch22Wrapper` — requires pycatch22

These are pure supervised regressors (not forecasters), so they use the standard
`train_single_model()` path from `model_trainer.py` with no special handling.

**Scaler policy:** Use `standard` scaler only (1 variant, same rationale as Plan 96-02).
The 3D-reshape in `SktimeRegressionWrapper` is scaler-invariant, and the scaler step
normalises the lag feature vector before reshaping — providing meaningful input regardless.

**Numba first-run caching:** The Docker build step should pre-warm numba by importing
ROCKET once during image build. This avoids JIT compilation delay at training time.

### 2. Add `train_sktime_regression_models()` to `model_trainer.py`

```python
def train_sktime_regression_models(
    X_train: np.ndarray,
    y_train: np.ndarray,
    X_test: np.ndarray,
    y_test: np.ndarray,
    n_splits: int = 5,
    scaler_variants: list[str] | None = None,
) -> list[TrainingResult]:
    """Train all sktime time series regression models.

    Models in this family use a 3D reshape of X internally via
    ``SktimeRegressionWrapper``, treating each sample's feature vector
    as a univariate time series of length n_features.

    Uses only ``standard`` scaler (1 variant) to keep training time linear
    with number of models.
    """
    if scaler_variants is None:
        scaler_variants = ["standard"]

    results: list[TrainingResult] = []
    for config in SKTIME_REGRESSION_MODELS:
        for scaler_variant in scaler_variants:
            logger.info(
                "Training sktime regression model",
                extra={"model": config.name, "scaler": scaler_variant},
            )
            try:
                result = train_single_model(
                    config=config,
                    X_train=X_train,
                    y_train=y_train,
                    X_test=X_test,
                    y_test=y_test,
                    scaler_variant=scaler_variant,
                    n_splits=n_splits,
                )
                results.append(result)
            except Exception as exc:
                logger.warning(
                    "sktime regression model training failed — skipping",
                    extra={"model": config.name, "error": str(exc)},
                )
    return results
```

def train_all_models(
    X_train, y_train, X_test, y_test,
    n_splits: int = 5,
    include_boosters: bool = True,
    include_sktime: bool = True,
    include_sktime_regression: bool = True,   # ← NEW
) -> list[TrainingResult]:
    results = []
    results.extend(train_linear_models(...))
    results.extend(train_tree_models(...))
    if include_boosters:
        results.extend(train_booster_models(...))
    results.extend(train_distance_neural_models(...))
    if include_sktime:
        results.extend(train_sktime_models(...))
    if include_sktime_regression:              # ← NEW
        results.extend(train_sktime_regression_models(
            X_train, y_train, X_test, y_test, n_splits=n_splits
        ))
    return results

### 5. Pre-warm Numba in Dockerfile

Add a Numba pre-warm step to the ML Dockerfile **builder stage** (after pip install):

```dockerfile
# Pre-compile ROCKET numba kernels so they are cached in the image layer
# This avoids JIT compilation delay on the first training run in Kubernetes
RUN python -c "
from sktime.regression.kernel_based import RocketRegressor
import numpy as np
X = np.random.randn(10, 1, 20)
y = np.random.randn(10)
RocketRegressor(rocket_transform='minirocket', num_kernels=100).fit(X, y)
print('Numba ROCKET kernels pre-compiled OK')
"
```

Place this step **after** `pip install -r requirements.txt` in the builder stage, so the
Numba cache is baked into the image and available at runtime without recompilation.

### 6. End-to-End Validation

Run a full training smoke test including regression models:

```bash
python -m ml.pipelines.training_pipeline \
    --tickers AAPL \
    --horizons 1 \
    --skip-shap \
    --registry-dir /tmp/sktime-regression-smoke \
    2>&1 | grep -E "(sktime regression|minirocket|rocket|timeseries_forest|random_interval|catch22|ERROR|WARNING)"
```

Verify all 5 appear in registry:

```bash
python -c "
from ml.models.registry import ModelRegistry
reg = ModelRegistry(registry_dir='/tmp/sktime-regression-smoke')
models = reg.list_models()
target = {'minirocket', 'rocket', 'timeseries_forest', 'random_interval', 'catch22'}
found = {m['model_name'] for m in models}
missing = target - found
if missing:
    print(f'MISSING from registry: {missing}')
    exit(1)
print('All 5 sktime regression models in registry: OK')
for m in sorted(models, key=lambda x: x.get('oos_rmse', 999)):
    if m['model_name'] in target:
        print(f\"  {m['model_name']:30s}  OOS RMSE={m.get('oos_rmse', '?')}\")
"
```
