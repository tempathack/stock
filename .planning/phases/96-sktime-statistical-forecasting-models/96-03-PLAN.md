## Goal
Update the ML container Dockerfile to install sktime dependencies, run a smoke-test
training run to confirm all 6 sktime models train successfully within the system's
resource constraints, and verify the trained models appear correctly in the MinIO model
registry alongside existing families.

## Context
After Plans 96-01 and 96-02, the code is in place. This plan:
1. Updates the ML Dockerfile so the container includes all sktime soft-deps
2. Runs a smoke test via the existing training pipeline (single ticker, reduced data)
3. Confirms models appear in model registry with correct metadata
4. Commits all changes and pushes so Argo CD can sync the updated CronJob

**Resource budget (from `cronjob-training.yaml`):** 4–16 CPU cores, 8–16 GB RAM.
sktime models are CPU-only and fit comfortably within this budget:
- NaiveForecaster / ExponentialSmoothing / Theta: < 1s each
- AutoETS: 2–10s
- AutoARIMA (pmdarima, stepwise): 5–30s per fit
- BATS: 30–120s with `n_jobs=-1` (parallelises across cores)

## Tasks

### 1. Update `ml/Dockerfile`

The Dockerfile uses a multi-stage build. In the builder stage where pip installs happen,
no change is needed — the new deps in `requirements.txt` are automatically picked up.

However, BATS uses Cython extensions and TBATS needs C compilation. Ensure the builder
stage has the required build tools:

```dockerfile
# In the builder stage (before pip install)
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*
```

If `build-essential` is already present (check existing Dockerfile), this task is a no-op.

In the runtime stage, add `libgomp1` if not already present (needed by statsforecast's
Numba JIT and lightgbm — likely already there):

```dockerfile
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq5 \
    libgomp1 \
    && rm -rf /var/lib/apt/lists/*
```

### 2. Smoke-Test Training Run (local)

Run a quick training smoke test directly in the ML container environment to validate
all 6 sktime wrappers train end-to-end through the full pipeline:

```bash
# In the ml container / local venv with deps installed
python -m ml.pipelines.training_pipeline \
    --tickers AAPL \
    --horizons 1 \
    --skip-shap \
    --registry-dir /tmp/sktime-smoke-test \
    2>&1 | tee /tmp/sktime-training.log
```

### 3. Verify Models Appear in Registry

After the smoke test, check that all 6 sktime models were saved to the registry:

```bash
python -c "
from ml.models.registry import ModelRegistry
reg = ModelRegistry(registry_dir='/tmp/sktime-smoke-test')
models = reg.list_models()
sktime_names = {'naive_last', 'exponential_smoothing', 'auto_ets',
                'theta', 'auto_arima', 'bats'}
found = {m['model_name'] for m in models}
missing = sktime_names - found
if missing:
    print(f'MISSING: {missing}')
else:
    print(f'All 6 sktime models registered: OK')
    for m in models:
        if m['model_name'] in sktime_names:
            print(f\"  {m['model_name']:30s}  OOS RMSE={m['oos_rmse']:.4f}\")
"
```

### 4. Verify No OOM / Timeout During BATS Training

BATS is the heaviest model. Confirm it completes within 5 minutes on real data
(500 samples is typical for a single ticker with 2 years of daily data):

```bash
python -c "
import numpy as np, time
from ml.models.sktime_wrappers import BATSWrapper

np.random.seed(42)
y = np.cumsum(np.random.randn(500)) + 100.0   # realistic price-like series
X = np.random.randn(500, 50)

start = time.time()
b = BATSWrapper(n_jobs=-1)
b.fit(X[:400], y[:400])
preds = b.predict(X[400:])
elapsed = time.time() - start
print(f'BATS completed in {elapsed:.1f}s, forecast={preds[0]:.4f}')
assert elapsed < 300, f'BATS too slow: {elapsed:.0f}s'
print('BATS timing: OK')
"
```
