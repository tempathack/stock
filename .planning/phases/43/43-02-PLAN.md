<objective>
Add horizon query parameter support to GET /predict/{ticker} and GET /predict/bulk so clients
can request predictions for specific horizons (1d, 7d, 30d). Add a GET /predict/horizons
discovery endpoint. All changes are backward-compatible — omitting the horizon parameter
returns the existing default (7d) behavior.

Purpose: ADVML-05 (GET /predict/{ticker}?horizon=1|7|30 query parameter support).
Output: API consumers can query any trained horizon. The /predict/horizons endpoint lets
clients discover which horizons are available before requesting predictions.
</objective>
