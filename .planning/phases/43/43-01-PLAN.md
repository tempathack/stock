<objective>
Add multi-horizon target generation (1d, 7d, 30d) to the label generator and integrate a
per-horizon training loop into the pipeline. Each horizon gets its own model suite trained,
ranked, and deployed independently. The single-horizon mode remains the default for full
backward compatibility.

Purpose: ADVML-03 (multi-horizon label generation) + ADVML-04 (separate model suite per horizon).
Output: The training pipeline can train 3× the model suites — one per horizon — each with its
own winner, registry directory, and serving artifacts. The deployer writes a horizons.json
manifest so downstream consumers know which horizons are available.
</objective>
