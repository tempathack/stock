# Plan 16-01 — SHAP Explainability: Analysis Module & Pipeline Component

## Objective
Implement `ml/evaluation/shap_analysis.py` — SHAP value computation with explainer selection matched to model type (TreeExplainer for tree/boosting, LinearExplainer for linear, KernelExplainer fallback for distance/neural). Implement `ml/pipelines/components/explainer.py` — pipeline component that orchestrates SHAP for the top-N models and stores results alongside model artifacts in the registry. Full test coverage. Satisfies **EVAL-11** and **EVAL-12**.

## Dependencies
- `ml/models/model_configs.py` — `TrainingResult`, `LINEAR_MODELS`, `TREE_MODELS`, `BOOSTER_MODELS`, `DISTANCE_NEURAL_MODELS`
- `ml/models/registry.py` — `ModelRegistry` (save/load artifacts, stores SHAP results in same folder)
- `ml/evaluation/ranking.py` — `rank_models()`, `RankedModel`
- `ml/pipelines/components/model_selector.py` — `select_and_persist_winner()` (saves top-5 models)

## Design Decisions

1. **Explainer routing by model family:**
   - **Tree-based** (`TREE_MODELS` + `BOOSTER_MODELS` names): `shap.TreeExplainer`
   - **Linear** (`LINEAR_MODELS` names): `shap.LinearExplainer`
   - **Distance / Neural** (`DISTANCE_NEURAL_MODELS` names — KNN, SVR, MLP): `shap.KernelExplainer`
   - Routing uses a lookup set built from `model_configs.py` dictionaries — no string heuristics.

2. **KernelExplainer budget:** Background data capped at `shap.kmeans(X_train, 50)`. Explanation data capped at 200 samples (random subsample if larger). Tree and Linear explainers operate on the full dataset — no cap.

3. **Storage format:** SHAP results stored **in the same registry folder** as `pipeline.pkl` and `metadata.json`:
   - `shap_importance.json` — list of `{"feature": str, "mean_abs_shap": float}` sorted descending
   - `shap_values.json` — full SHAP value matrix (list of lists) + feature names + sample count; enables beeswarm rendering on the frontend

4. **Top-N scope:** Compute SHAP for the top 5 ranked models (matching the top-5 persisted by `model_selector.py`).

      **Imports:**
      ```python
      import logging
      import numpy as np
      import shap
      from sklearn.pipeline import Pipeline
      from ml.models.model_configs import LINEAR_MODELS, TREE_MODELS, BOOSTER_MODELS
      ```

      **Constants:**
      ```python
      _TREE_NAMES: set[str]   # union of TREE_MODELS.keys() and BOOSTER_MODELS.keys()
      _LINEAR_NAMES: set[str] # LINEAR_MODELS.keys()
      # Everything else → kernel fallback
      ```

      **Functions:**

      ```python
      def get_explainer_type(model_name: str) -> str:
      ```
      - Returns `"tree"` if `model_name` in `_TREE_NAMES`
      - Returns `"linear"` if `model_name` in `_LINEAR_NAMES`
      - Returns `"kernel"` otherwise (fallback for KNN, SVR, MLP, or unknown)

      ```python
      def compute_shap_values(
          pipeline: Pipeline,
          X: np.ndarray,
          model_name: str,
          feature_names: list[str],
          max_samples: int = 200,
          background_samples: int = 50,
      ) -> tuple[np.ndarray, list[str]]:
      ```
      - Extracts the fitted model from pipeline's last step (named `"model"`)
      - Extracts the scaler from pipeline (if present) and transforms X through it
      - Determines explainer type via `get_explainer_type(model_name)`
      - For `"tree"`: `shap.TreeExplainer(model).shap_values(X_transformed)` — full dataset
      - For `"linear"`: `shap.LinearExplainer(model, X_transformed).shap_values(X_transformed)` — full dataset
      - For `"kernel"`:
        - Background: `shap.kmeans(X_transformed, min(background_samples, len(X_transformed)))``
        - If `len(X_transformed) > max_samples`: randomly subsample X_transformed to max_samples rows
        - `shap.KernelExplainer(model.predict, background).shap_values(X_subset)`
      - Returns `(shap_values_array, feature_names)` — shap_values shape is (n_samples, n_features)

      ```python
      def compute_feature_importance(
          shap_values: np.ndarray,
          feature_names: list[str],
      ) -> list[dict]:
      ```
      - Computes `mean(|shap_values|, axis=0)` per feature
      - Returns list of `{"feature": str, "mean_abs_shap": float}` sorted descending by importance

      ```python
      def get_shap_summary(
          pipeline: Pipeline,
          X: np.ndarray,
          model_name: str,
          feature_names: list[str],
          max_samples: int = 200,
      ) -> dict:
      ```
      - Calls `compute_shap_values()` then `compute_feature_importance()`
      - Returns:
        ```python
        {
            "shap_values": shap_values.tolist(),  # for JSON serialization
            "feature_names": feature_names,
            "n_samples": len(shap_values),
            "feature_importance": importance_list,
        }
        ```

      def explain_top_models(
          results: list[TrainingResult],
          pipelines: dict[str, Pipeline],

          feature_names: list[str],
          registry_dir: str = "model_registry",
          top_n: int = 5,

        6. Write `shap_values.json` — the full summary dict (shap_values, feature_names, n_samples, feature_importance)
        7. Log completion for each model
      - Returns dict keyed by model_key → `{"feature_importance": [...], "shap_summary": {...}}`
