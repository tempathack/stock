# Phase 23 — FastAPI Prediction & Model Endpoints — Context

## Requirements

- **API-07**: GET /predict/{ticker} → 7-day forecast
- **API-08**: GET /predict/bulk → forecasts for all S&P 500
- **API-09**: GET /models/comparison → model metrics comparison
- **API-10**: GET /models/drift → drift detection status

## Existing Infrastructure

### FastAPI Service (`services/api/`)
- main.py: lifespan app with health + ingest routers registered
- config.py: Settings with SERVICE_NAME, DATABASE_URL, KAFKA, TICKER_SYMBOLS etc.
- routers/: health.py (done), ingest.py (done), predict.py (stub), models.py (stub), market.py (stub)
- services/: prediction_service.py (stub), market_service.py (stub)
- models/schemas.py: IngestRequest, IngestResponse, ErrorResponse

### ML Backend
- `ml/models/registry.py`: ModelRegistry with list_models(), get_winner(), get_active_model(), load_model()
- `ml/pipelines/components/predictor.py`: generate_predictions(data_dict, serving_dir, horizon), save_predictions()
- `ml/drift/trigger.py`: DriftLogger with get_recent_events()
- `ml/drift/detector.py`: DriftResult dataclass
- `ml/evaluation/ranking.py`: RankedModel, WinnerResult, rank_models()

### DB Schema (reference)
- predictions: id, ticker, prediction_date, predicted_date, predicted_price, model_id, confidence, created_at
- model_registry: model_id, model_name, version, metrics_json, trained_at, is_active
- drift_logs: id, drift_type, severity, details_json, detected_at

## Design Approach

For Phase 23, endpoints will read from the **file-based** model registry and drift logs (no DB connection required). This matches the existing Phase 20-22 architecture where ModelRegistry and DriftLogger are file-based.

The `/predict/{ticker}` and `/predict/bulk` endpoints will read cached predictions from `model_registry/predictions/latest.json` (generated by the predictor component). If no cached predictions exist, return 404.

The `/models/comparison` endpoint will read all model metadata from the file-based registry.

The `/models/drift` endpoint will read recent drift events from the JSONL drift log.
