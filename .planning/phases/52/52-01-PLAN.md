    # --- ml/models/storage_backends.py (NEW) ---
    - Module providing StorageBackend protocol and two concrete implementations

    # StorageBackend Protocol:
    - typing.Protocol class with 5 abstract methods:
      - write_bytes(key: str, data: bytes) -> None
      - read_bytes(key: str) -> bytes
      - write_json(key: str, obj: Any) -> None
      - read_json(key: str) -> Any
      - list_keys(prefix: str) -> list[str]
      - delete(key: str) -> bool
      - exists(key: str) -> bool

    # LocalStorageBackend:
    - __init__(base_dir: str) → creates Path(base_dir), mkdir parents=True exist_ok=True
    - write_bytes: writes data to base_dir / key, creating parent dirs as needed
    - read_bytes: reads and returns bytes from base_dir / key; raises FileNotFoundError if missing
    - write_json: json.dumps(obj, indent=2, default=str).encode() → write_bytes
    - read_json: json.loads(read_bytes(key).decode())
    - list_keys(prefix): glob base_dir / prefix / "**/*", return relative POSIX paths (files only)
    - delete(key): if file exists, unlink and prune empty parents up to base_dir; return True. If dir, shutil.rmtree. Else return False
    - exists(key): return (base_dir / key).exists()
    - Behavior must be exactly equivalent to current ModelRegistry filesystem operations

    # S3StorageBackend:
    - __init__(bucket: str, prefix: str = "", endpoint_url: str | None = None, region_name: str = "us-east-1")
    - Creates boto3 S3 client using:
      - endpoint_url from param (defaults to MINIO_ENDPOINT env var)
      - aws_access_key_id from MINIO_ROOT_USER env var (or AWS_ACCESS_KEY_ID)
      - aws_secret_access_key from MINIO_ROOT_PASSWORD env var (or AWS_SECRET_ACCESS_KEY)
      - region_name from param
    - Credential resolution order: explicit params → MINIO_ROOT_USER/PASSWORD → AWS_ACCESS_KEY_ID/SECRET_ACCESS_KEY (boto3 default chain)
    - All keys are prefixed with self._prefix if non-empty: f"{prefix}/{key}" (strip leading/trailing slashes)
    - write_bytes: s3_client.put_object(Bucket=bucket, Key=full_key, Body=data)
    - read_bytes: s3_client.get_object(...)["Body"].read(); catch ClientError NoSuchKey → raise FileNotFoundError
    - write_json: json.dumps(obj, indent=2, default=str).encode() → write_bytes
    - read_json: json.loads(read_bytes(key).decode())
    - list_keys(prefix): paginate list_objects_v2 with Prefix=full_prefix, return keys with self._prefix stripped
    - delete(key): s3_client.delete_object(...); return True. Catch ClientError → return False
    - exists(key): s3_client.head_object(...), return True; catch ClientError 404 → return False
    - All S3 calls wrapped with minimal error handling (let unexpected errors propagate)

    # Factory function:
    - create_storage_backend(backend_type: str = "local", **kwargs) -> StorageBackend
    - backend_type read from STORAGE_BACKEND env var if not explicitly passed
    - "local" → LocalStorageBackend(base_dir=kwargs.get("base_dir", "model_registry"))

    - Raises ValueError for unknown backend_type
    - This satisfies OBJST-08
