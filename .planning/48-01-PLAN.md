# 48-01-PLAN — slowapi Rate Limiting

## Goal
Add per-endpoint rate limiting using slowapi with configurable limits via environment variables.

## Changes

### 1. `services/api/requirements.txt`
- Add `slowapi>=0.1.9`

### 2. `services/api/app/config.py`
- Add rate-limit env vars:
  - `RATE_LIMIT_GLOBAL` (default "100/minute")
  - `RATE_LIMIT_PREDICT` (default "30/minute")
  - `RATE_LIMIT_INGEST` (default "10/minute")

### 3. `services/api/app/rate_limit.py` (NEW)
- Create `Limiter` instance using `slowapi.Limiter` with `get_remote_address` key function
- Custom `_rate_limit_exceeded_handler` returning 429 with `Retry-After` header
- Export `limiter` singleton

### 4. `services/api/app/main.py`
- Import and attach limiter state: `app.state.limiter = limiter`
- Add `RateLimitExceeded` exception handler
- Health endpoint is exempt (no decorator)

### 5. Router decorators
- `predict.py`: `@limiter.limit(settings.RATE_LIMIT_PREDICT)` on all endpoints
- `ingest.py`: `@limiter.limit(settings.RATE_LIMIT_INGEST)` on all endpoints
- All other routers: `@limiter.limit(settings.RATE_LIMIT_GLOBAL)` as default

### 6. `k8s/ingestion/configmap.yaml`
- Add `RATE_LIMIT_GLOBAL`, `RATE_LIMIT_PREDICT`, `RATE_LIMIT_INGEST`

## Success Criteria
- 101st global request in 1 minute returns 429 with Retry-After
- /predict/* capped at 30/min per IP
- /ingest/* capped at 10/min per IP
- /health is exempt from rate limiting
